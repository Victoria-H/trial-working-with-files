Documentation du Job

Vue d'ensemble

Vous venez de passer du temps sur la phase de planification. Cette section va couvrir rapidement les bonnes pratiques relatives à la phase de développement. Plus précisément, les propriétés des Job, les noms et libellés des composants, ainsi que l'utilisation de notes dans l'espace de modélisation graphique. Ensuite, vous allez passer à la phase de documentation afin de générer et voir la documentation HTML pour un Job précédemment créé. Comme dans tout projet, une bonne documentation est importante et peut permettre aux autres intervenants de comprendre un Job Talend.

Amélioration des propriétés clés d'un Job

  1. Ouvrez le Job WeatherWebService et cliquez sur sa vue Job :

    Ces informations sont celles que vous avez fournies à la création du Job. Pour éviter de perdre du temps, vous n'avez pas documenté sérieusement les champs Purpose et Description de ce Job et des autres. Ce n'est pas une bonne pratique, vous devez corriger cela.

  2. Saisissez un texte plus utile dans le champ Purpose. Par exemple Get weather by ZIP code.

  3. Saisissez un texte plus utile dans le champ Description. Par exemple Send a ZIP code to a public SOAP web service which returns the current weather.
  4. Sauvegardez les modifications de votre Job.

Notez que vous pouvez également modifier les propriétés de ces Jobs sans même ouvrir le Job. Cliquez-droit sur le Job dans le référentiel et sélectionnez Edit properties :

 

Ajout d'une Note

  1. Ajoutez un composant Note dans l'espace de modélisation graphique de votre Job. Même si vous pouvez le trouver dans le dossier Misc, il est plus simple de saisir note n'importe où dans l'espace de modélisation graphique.

  2. Double-cliquez sur le composant et, dans sa vue Component, saisissez votre texte dans le champ Text. Considérez une convention de haut niveau comprenant les informations de base et le formatage, par exemple :

    Notez que le texte ne revient pas automatiquement à la ligne, vous devez donc saisir le texte sur différentes lignes. Il y a différentes options de formatage. Si vous avez le temps et que cela vous intéresse, amusez-vous avec les options pendant quelques minutes.

  3. Glissez-déposez la Note en haut à gauche du Job. Redimensionnez le composant Note, si nécessaire, pour l'ajuster au texte saisi. Votre Job doit ressembler à celui-ci : 

     

  4. Placez une autre Note dans l'espace de modélisation graphique, sous le Job. Une autre suggestion pour une bonne pratique durant la phase de développement est de placer une liste d'élements à faire. Vous pouvez modifier la couleur de la note, elle pourrait ressembler à ceci :

    5. Bien sur, le contenu de la Note n'est pas important dans l'exemple ci-dessus.

Documentation d'un composant

Jusqu'ici, vous avez placé des composants dans l'espace de modélisation graphique, laissé leur nom par défaut et fourni peu ou pas d'information(s). Cette section vous présente les bonnes pratiques relatives aux noms de composants et autres informations descriptives utiles.

  1. Ouvrez la vue Component du tESBConsumer. Cliquez sur l'onglet View :

  2. Saisissez le texte suivant dans le champ Label format : <b>__UNIQUE_NAME__</b><br>Invoke Get Weather method.

    3. D'un coup d'oeil sur l'espace de modélisation graphique, vous pouvez voir l'icône du composant, son nom (en gras) et un bref rappel de sa fonction. Toutes ces informations peuvent être utiles, particulièrement pour les nouveaux développeurs ou les autres intervenants confrontés au Job.

  3. Dans le champ Hint format, sous Label format, saisissez un contenu plus utile, comme : Invoke public SOAP weather service using the GetWeather method with a ZIP code. Notez que le composant tESBConsumer ne reflète pas la modification (l'étape suivante vous permettra d'activer le champ Hint format).
  4. Passez à la vue Documentation du tESBConsumer. Cochez la case Show Information :

  5. Saisissez d'autres informations utiles aux gens qui voudraient en savoir plus sur ce composant, dans le champ Comment, par exemple : The full method (operation) name for the get request is GetCityWeatherByZIP. Response is GetCityWeatherByZIPResponse. Notez que cette information s'affiche lorsque vous verrez et générerez la documentation HTML dans la section suivante (puisque vous êtes dans la vue Documentation, saisissez maintenant les informations).
  6. Passez votre curseur sur l'icône d'information au-dessus du tESBConsumer, dans l'espace de modélisation graphique pour voir l'astuce (champ Hint format) s'afficher :

  7. (Si le temps et votre intérêt le permettent) Répétez ces étapes pour mieux documenter un autre composant, par exemple l'un des composants tXMLMap.

  8. Sauvegardez le Job.

Visualisation et génération de la documentation HTML

Avant de générer la documentation et de la visualiser depuis le système de fichiers, vous pouvez la voir directement dans le Studio. Cette fonctionnalité est pratique pour vérifier rapidement à quoi ressemble votre HTML durant le cycle de développement, qui peut comprendre de nombreuses itérations.

  1. Dans le Repository, cliquez-droit sur le WeatherWebService et sélectionnez  View documentation :

    Un nouvel onglet s'ouvre, contenant un aperçu de la documentation HTML.

  2. Vous aurez bientôt plus de temps pour lire la documentation. Pour le moment, cliquez simplement le l'hyperlien Component List.
  3. Sélectionnez tESBConsumer dans la liste des composants. Remarquez la liste Component Parameters, présentant les paramètres détaillés, en faisant défiler vers le bas. Vers le bas des paramètres, vous devriez voir la propriété Comment, associée à sa valeur (Value), qui correspond au texte saisi dans le champ Comment, au cours de l'exercice :

    4. Maintenant que la documentation au sein du Studio semble correcte, vous pouvez en générer une version plus permanente.

  4. Cliquez-droit sur le Job WeatherWebService dans le Repository et sélectionnez Generate Doc As HTML :

  5. Parcourez votre système de fichiers jusqu'au dossier C:\StudentFiles, utilisez la valeur par défaut dans le champ File name puis cliquez sur Save lorsque vous avez terminé :

  6.  La boîte de dialogue Generate Documentation doit ressembler à ceci :

  7. Trouvez l'archive dans la structure du répertoire et extrayez le contenu à l'aide de 7-Zip (Astuce : cliquez-droit sur le fichier et sélectionnez > 7-Zip > Extract here).

  8. Trouvez et ouvrez le fichier WeatherWebServices_0.1.html dans le dossier WeatherWebService_0.1. Examinez durant quelques minutes la documentation créée. Vous pouvez constater que tous vos commentaires sont inclus, ainsi que les paramètres des composants et de nombreuses autres données :
