Création d'un Business Model

Construction d'un Business Model

  1. Dans le Repository, cliquez-droit sur Business Models et sélectionnez Create Business Model :

  2. Saisissez CustCleanup dans le champ Name et le texte approprié dans les champs Purpose et Description. Cliquez sur Finish :

    Seul le nom est obligatoire, mais une bonne documentation est importante dans les Business Models, les bonnes pratiques vous recommandent d'être rigoureux.

  3. Le nom du nouveau Business Model s'affiche dans le Repository, un espace de modélisation graphique s'ouvre, vide, puis la Palette change :

    À la place des composants pour les Jobs, la Palette affiche des formes que vous pouvez utiliser pour construire votre Business Model :

Ajout de formes

  1. Dans la Palette, cliquez sur Input et cliquez dans l'espace de modélisation graphique pour placer la forme.
    Saisissez Customer Data dans la boîte de texte :

    Cette forme représente le fichier d'entrée des données clients. Astuce : si nécessaire, utilisez les poignées pour élargir la forme afin que le nom soit entièrement contenu dans la forme.

  2. Ajoutez une forme Data et nommez-la State Codes.
    La deuxième forme représente la table de référence des codes et noms des États.

  3. Placez votre curseur sur une zone vide de l'espace de modélisation graphique et immobilisez-le jusqu'à ce que la barre d'accès rapide s'affiche :

    Cette barre d'outils vous permet de placer des formes sans utiliser la Palette.

  4. Cliquez sur la forme Action dans la barre d'outils et nommez-la Join, puis déplacez-la près des autres formes :

La forme Join représente le composant tMap qui effectue le mapping et les jointures.

Connexion des formes

  1. Placez votre curseur sur le côté droit de la forme Customer Data jusqu'à ce que deux flèches (entrée/sortie) s'affichent :

    Glissez la flèche pointant vers la droite, jusqu'à la forme Join et cliquez sur Create Relationship :

  2. Vous pouvez utiliser un moyen alternatif afin de connecter vos formes. Note : la relation affichée dans le diagramme ci-dessus (et dessous) n'est pas directionnelle. Vous pouvez sélectionner Create Directional Relationship si vous souhaitez un flux de données plus précis.

    Placez votre curseur sur State Codes. Une fois les deux flèches affichées, déposez la flèche pointant vers la droite n'importe où dans l'espace de modélisation graphique. Relâchez la flèche. Un menu contextuel s'affiche. Cliquez sur Create Relationship To puis sur Existing element :

  3. Dans la liste, sélectionnez Action Business Item Join puis cliquez sur OK :

  4. Une fenêtre s'ouvre et vous demande de confirmer la connexion. Cliquez sur Yes :

  5. Ajoutez une forme Terminal, nommez-la End et reliez la forme Join à celle-ci. Si vous avez opté pour des relations directionelles, votre Business Model doit ressembler à ceci :

    Note : les formes ont des noms représentant leur but suggéré, comme Action ou Terminal, mais vous pouvez les utiliser comme vous le souhaitez. Il n'y a aucune fonctionnalité derrière les formes et les connexions. Elles servent uniquement à modéliser et planifier. Le modèle construit ici est relativement simple, mais il pourrait être plus élaboré.

  6. Sélectionnez la forme Join puis ouvrez la vue Business Models sous l'espace de modélisation graphique.

    Vous pouvez personnaliser l'apparence des formes, la taille du texte, la couleur, l'alignement, la police, etc.

  7. Expérimentez les différentes modifications que vous pouvez apporter à l'apparence de la forme Join.

Ajout d'une Note

  1. Pour ajouter une note à votre Business Model, cliquez-droit dans l'espace de modélisation graphique et sélectionnez Add > Note :

  2. La Note s'affiche, placez-la dans votre modèle.

    L'un des objectifs d'un Business Model est de documenter un projet et la note est l'un des mécanismes de documentation.

  3. Remplacez le texte par défaut par un message concernant le projet. Notez que la forme peut s'agrandir pour s'ajuster au texte.

  5. Sauvegardez le Business Model.
