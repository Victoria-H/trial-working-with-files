Mise à jour du Business Model

Vue d'ensemble

Un Business Model est utile lorsqu'il est à jour, pensez donc à l'ouvrir régulièrement afin de vérifier que toutes les parties sont correctement représentées. Au cours de cette section, vous allez ajouter de la documentation externe depuis votre système de fichiers local dans le référentiel et l'inclure en tant que partie de votre Business Model.

Ajout d'un plan de projet

  1. Dans le Repository, cliquez-droit sur Documentation et sélectionnez Add documentation :

    Vous pouvez ainsi ajouter des fichiers de documentation à votre projet.

  2. Cliquez sur Browse et trouvez le fichier ProjectPlan.txt dans C:/StudentFiles. Saisissez du texte approprié dans les champsPurpose et Description :

    3. Cliquez sur Finish lorsque vous avez terminé.

  3. Dans le Repository, cliquez-droit sur Documentation et sélectionnezAdd documentation :

  4. Cliquez sur Browse et sélectionnez le fichier C:/StudentFiles/ProjectPlan_Input.txt, saisissez DataInput dans le champ Name et renseignez les champs Purpose etDescription :

    Cliquez sur Finish lorsque vous avez terminé.

Association d'éléments

  1. Ouvrez le Business Model CustCleanup précédemment créé.

  2. Placez une forme Document à gauche, nommez-la Project Plan :

  3. Déposez ProjectPlan.txt du Repository dans la forme Project Plan :

    Astuce : lorsque vous passez votre curseur sur votre projet dans le noeud Documentation dans le Repository, la Description que vous avez ajoutée précédemment s'affiche.

  4. Dans la vue BusinessModels sous l'espace de modélisation graphique, cliquez sur l'onglet Assignment :

    5. Cette vue vous indique ce qui est associé à une forme spécifique et vous permet d'ajouter un commentaire.

  5. Dans Repository > Job Designs > Standard > Repository > Job Designs > Standard > Customers, déposez LookupState sur la forme Join, puis cliquez sur l'onglet Assignment :

    Vous pouvez assigner presque n'importe quel élément du référentiel à une forme dans un Business Model. Vous pouvez assigner différents éléments à une forme et assigner le même élément à différentes formes. Cela simplifie l'utilisation du Business Model comme emplacement central pour documenter et accéder au projet.

  6. Déposez DataInput depuis le Repository > Documentation sur les formes Customer Data et State Codes. Cliquez-droit sur DataInput sous Documentation et sélectionnez Find Assignments :

    Vous pouvez voir que ce document a deux affectations, les formes sont entourées d'une bordure orange épaisse. Cet bordures ne sont pas permanentes, elles ne sont pas ajoutées au Business Model. Elles servent uniquement à vous indiquer les affectations du document sélectionné.

  7. Dans Repository > Documentation, cliquez-droit sur ProjectPlan.txt sous Documentation, puis sélectionnez Find Assignments :

    Le document Project Plan est entouré d'une bordure orange et les documents Customer Data et State Codes n'en ont pas.

    Astuce : si le nom d'un document n'est pas très descriptif, vous pouvez le voir directement depuis le Studio. Pour ce faire, double-cliquez sur ProjectPlan.txt dans le référentiel pour ouvrir une copie en lecture seule du document dans l'espace de modélisation graphique.

  8. Sauvegardez le Business Model.

Étape suivante

Vous allez suivre les bonnes pratiques relatives aux conventions de nommage des Jobs et composants, ainsi que pour les astuces.
