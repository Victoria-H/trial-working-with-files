Création de la table Products

Vue d'ensemble

L'étape suivante consiste à créer et alimenter la table des produits.

Duplication du Job

  1. Dupliquez le Job Custprep2Table, nommez-le ProductPrep2Table.
    Lorsque vous créez un Job très similaire à un Job existant, cela peut vous faire gagner du temps de le dupliquer plutôt que de le créer à partir de rien.

  2. Ouvrez le nouveau Job et fermez tous les autres Jobs, pour éviter la confusion.

Création et alimentation de la table Products

  1. Double-cliquez sur le composant tCreateTable.
    Dans la liste Schema, sélectionnez RetailProducts - ProductMetadata.
    Modifiez la valeur du champ Table Name en "RetailProducts" :

  2. Double-cliquez sur le RetailCustomers pour ouvrir sa vue Component.
    Cliquez sur le bouton [...] à côté de la liste Property Type et sélectionnez RetailProducts dans les métadonnées de fichier délimité :

    Cliquez sur Yes dans la boîte de dialogue qui s'ouvre :

  3. Sélectionnez le schéma RetailProducts - ProductMetadata.

    Répondez Yes lorsqu'il vous est proposé de propager les modifications.

  4. Dans l'onglet View, modifiez le nom du composant en RetailProducts :

  5. Double-cliquez sur le composant LocalMySQL pour ouvrir sa vue Component :

    Le schéma est maintenant Built-in, car vous avez propagé les modifications du schéma depuis le composant RetailProducts.

  6. Modifiez le type de schéma à Repository, sélectionnez le schéma RetailProducts - ProductMetadata et modifiez le nom de la table en "RetailProducts" :

    Ce composant va écrire dans une table nommée RetailProducts avec le schéma pris depuis le Repository.

  7. Exécutez le Job. Le fichier des produits contient 50 lignes :

  8. Notez qu'un avertissement s'affiche sur le tCreateTable et le LocalMySQL. Passez votre curseur sur le point d'exclamation, pour lire les avertissements :
  9. Pour résoudre l'avertissement, double-cliquez sur le composant tCreateTable.
    Cliquez sur le bouton [...] à côté du champ Edit Schema, sélectionnez Change to built-in property puis cliquez sur OK :

  10. Le schéma du tCreateTable s'affiche.
    Notez que la colonne DB Type est vide.
    Cliquez sur le bouton Reset DB Types et cliquez sur OK :

  11. La colonne DB Type est renseignée :

  12. Cliquez sur OK et notez que le point d'exclamation n'est plus affiché :

  13. L'avertissement sur le LocalMySQL est également relatif au fait que le type DB Type n'est pas défini dans le schéma. Pour supprimer l'avertissement sur ce composant, répétez les étapes que vous venez d'effectuer pour supprimer l'avertissement sur le tCreateTable.

Vérification des résultats

  1. Dans MySQL Workbench, remplacez le texte existant dans la fenêtre Query par la commande suivante :

    select * from retailproducts;

    Cliquez sur l'icône représentant un éclair, afin de vérifier les résultats :

    Si la requête ne retourne pas de résultat, vérifiez votre travail et effectuez les corrections nécessaires.
