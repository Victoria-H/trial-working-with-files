Création d'une métadonnée de base de données

Vue d'ensemble

Vous pouvez créer une métadonnée relative à la connexion au serveur local de votre base de données et à votre base de données spécifique. Créer des métadonnées avec ces informations simplifie la réutilisation dans différents composants et Jobs.

Création d'une métadonnée de connexion à une base de données

  1. Dans le Repository, sous Metadata, cliquez-droit sur Db Connections et sélectionnez Create connection :

    Note : Comme de nombreux composants dans votre projet doivent se connecter à la même base de données, stocker les informations de connexion dans le Repository vous permet de ne pas saisir ces informations pour chaque composant.

  2. Saisissez LocalMySQL dans le champ Name, renseignez les champs Purpose et Description. Cliquez sur Next :

Configuration de la métadonnée de connexion à la base de données

  1. Sélectionnez MySQL dans la liste déroulante DB Type. Configurez la connexion comme suit :
    Login : root
    Password : root
    Server : localhost
    DataBase : training

    Cliquez sur le bouton Check :

    Note : ces informations de configuration sont spécifiques à l'environnement de formation, dans lequel une base de données MySQL est configurée pour vous. Dans votre propre environnement, vous devez fournir les informations de configuration appropriées à votre configuration.

  2. La vérification de la connexion doit réussir. Cliquez sur OK :

    Si la connexion ne s'établit pas, corrigez les informations de connexion et réessayez.

    3. Note : la première fois que vous configurez une connexion dans votre propre environnement, il vous est proposé de télécharger et installer un fichier .jar externe. Si vous voyez cette boîte de dialogue, installez le fichier et continuez. Tous les fichiers .jar requis sont déjà installés dans l'environnement de formation.

Export en tant que contexte

Comme vous l'avez fait pour la métadonnée RetailCustomer, vous allez créer des variables de contexte avec toutes les informations relatives à votre base de données MySQL.

  1. Cliquez sur Export as Context puis sur Next :

  2. Dans l'assistant Create / Reuse a context group, cliquez sur Next :

  3. Dans l'onglet Variables, vous allez voir la liste des variables de contexte créées et, dans l'onglet Value as Table, vous allez voir la valeur par défaut de chaque variable de contexte.
    Cliquez sur Finish :

  4. Retournez dans la fenêtre de connexion à la base de données, notez que, au lieu des valeurs saisies dans chaque champ, vous voyez des variables de contexte.
    Cliquez sur Finish :

Étape suivante

Maintenant que les informations de connexion à votre base de données MySQL locales sont configurées, vous pouvez créer la première table de base de données.
