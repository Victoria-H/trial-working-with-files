Démarrage du Studio Talend

Vue d'ensemble

Avant de commencer à développer dans le Studio Talend, vous devez vous familiariser avec son environnement. La première étape consiste à démarrer le logiciel et à ouvrir un projet existant . Vous aller découvrir la fenêtre principale du Studio Talend et ses barres d'outils.

Ouverture d'un projet dans le Studio Talend

  1. Double-cliquez sur l'icône suivante sur votre bureau pour exécuter le Studio Talend :

    La méthode alternative consiste à savoir, dans le système, où sont situés les fichiers binaires, afin de pouvoir les exécuter directement. Par exemple, naviguez jusqu'à C:\Talend\Numérodeversion\studio. Vous pouvez constater qu'il y a plusieurs exécutables, correspondants aux différentes architectures système. L'environnement de formation est sous Windows, 64-bits, donc double-cliquez sur Talend-Studio-win-x86_64.exe pour démarrer le Studio. Cela est utile lorsqu'aucun raccourci n'existe. Vous pouvez également créer un raccourci sur votre Bureau à partir d'ici.

  2. La fenêtre Talend Data Fabric s'ouvre. Sélectionnez la connexion Local, sélectionnez Select an existing project et sélectionnez le projet DI_B  :

    Le projet a déjà été créé pour vous.

    Il y a différents fichiers exécutables pour les différentes combinaisons de systèmes d'exploitation et de processeurs. Lorsque vous exécutez un logiciel Talend au travail ou à la maison, sélectionnez le fichier approprié à votre environnement. Dans cet environnement de formation, Talend Data Fabric est installé.

  3. Cliquez sur Finish pour ouvrir le projet.
    La fenêtre principale du projet s'ouvre directement.

Présentation de la fenêtre principale du Studio Talend

Avant de commencer à construire votre premier Job, regardez les différentes parties de l'interface du Studio.

Pour le moment, les panneaux et vues de votre environnement sont vides, comme aucun Job n'a été créé.
Voici à quoi ressemble votre environnement, une fois le premier Job construit dans la section suivante :


Utilisez cette image pour naviguer dans les panneaux et vues de la fenêtre principale :

  1. À gauche de la fenêtre, vous pouvez voir la vue Repository, contenant la structure du projet avec différents noeuds :
    • Le noeud Job Designs contient tous les Jobs développés, pouvant également être organisés en dossiers. Un Job est un processus graphique, comprenant un ou plusieurs composants reliés, vous permettant de configurer et exécuter des processus de gestion de flux de données. Un Job traduit des besoins métier et code, routines et programmes. En d'autres mots, il implémente techniquement votre flux de données.
    • Le dossier Contexts regroupe les fichiers contenant les variables de contexte que vous souhaitez réutiliser dans différents Jobs, comme les chemins d'accès ou les informations de connexion aux bases de données.
    • Le dossier Metadata stocke dans le référentiel des informations réutilisables relatives à des fichiers, bases de données et/ou systèmes auxquels vous devez accéder au sein de vos Jobs.
  2. À droite, vous pouvez voir la Palette et l'utiliser pour glisser-déposer des composants techniques dans l'espace de modélisation graphique, afin de construire votre Job.

  3. Au milieu, vous pouvez voir la vue Designer, l'espace de modélisation graphique. Vous allez utiliser cette zone pour construire vos Jobs de manière graphique en ajoutant des composants et en les connectant.

    Sous l'espace de modélisation graphique, vous pouvez accéder à différents onglets :

    • l'onglet Designer - qui s'ouvre par défaut lorsque vous créez un Job. Il affiche le Job en mode graphique.
    • l'onglet Code - qui vous permet de visualiser le code et de montrer les possibles erreurs de langage.
    • l'onglet Jobscript - qui vous permet de visualiser et modifier les Jobscripts.

  4. Dans la partie inférieure de l'espace de modélisation graphique, vous pouvez voir plusieurs vues.

    La vue Component affiche les propriétés de l'élément sélectionné dans l'espace de modélisation graphique. Ces propriétés peuvent être modifiées pour configurer les paramètres relatifs à un composant particulier.



    La vue Run est utilisée pour l'exécution des Jobs et l'exploration des résultats. Les résultats s'affichent dans la console, dans la zone Execution :

Présentation des barres d'outils du Studio Talend

  1. En haut de la fenêtre principale de Talend Data Fabric, différentes barres d'outils regroupent les fonctionnalités les plus utilisées de Talend  :
    • le menu principal
    • une barre d'accès rapide
    • les boutons pour passer d'une perspective à une autre :


    Selon la taille de la fenêtre du Studio Talend, la barre d'outils permettant de passer d'une perspective à une autre peut être affichée dans le coin supérieur droit, juste après la barre d'accès rapide, ou à gauche, sous la barre d'accès rapide.

  2. Une Perspective définit l'ensemble des vues et de leur disposition dans la fenêtre. La perspective sélectionnée est toujours celle active.

    Le Studio Talend Data Fabric correspond à la plateforme unifiée et sera utilisé pour créer des projets d'intégration de données, mais aussi pour d'autres projets : Big Data, intégration d'applications, Master Data Management ou profiling de données. C'est la raison pour laquelle différentes perspectives sont disponibles dans votre environnement de formation.

    Au cours de cette formation, seule la perspective Integration sera utilisée. Elle est la perspective par défaut lors de l'ouverture du projet DI_B.

  3. Si vous cliquez et ouvrez une autre perspective, vous pouvez toujours revenir à la perspective Integration en utilisant l'une des options suivantes :

    Option 1 : cliquez sur le bouton Integration dans la barre d'outils.

    Option 2 : dans le menu, cliquez sur Window et sélectionnez Reset Perspective....

    Lorqu'une boîte de dialogue vous demande si vous voulez réinitialiser la perspective, cliquez sur Yes.

    La perspective par défaut de votre environnement est la perspective Integration.

    Si vous réinitialisez la perspective, la perspective par défaut s'ouvre avec toutes les vues dans leur position initiale.

  4. Dans la barre d'outils du menu, sélectionnez Help > Help Contents :

    Cela ouvre la fenêtre Help - Talend Studio.

    Consultez cette section pour obtenir plus d'informations concernant les fonctionnalités du Studio Talend.

Vous avez ouvert le projet de formation et découvert l'interface du Studio Talend. Vous pouvez construire votre Premier Job.
