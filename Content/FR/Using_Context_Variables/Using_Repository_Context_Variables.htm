Utilisation des variables de contexte du Repository

Vue d'ensemble

Dans l'exercice précédent, vous avez créé une variable de contexte Built-In disponible uniquement dans un Job. Vous allez créer une variable de contexte dans le Repository pouvant être réutilisée dans n'importe quel Job du projet.

Création d'une variable de contexte du Repository

  1. Dans le Repository, cliquez-droit surContexts et sélectionnez Create context group :

    Note : un groupe de contextes est un outil pour organiser les variables du référentiel.Il est donc préférable de le nommer de manière explicite.

  2. Saisissez LocationContextVariables dans le champ Name et saisissez les informations appropriées dans les champs Purpose et Description.
    Cliquez sur Next :

  3. Cliquez sur le bouton [+] en bas à gauche de la table pour ajouter une variable. Nommez-la RepoOutputDir afin de la différencier par son nom de la variable de contexte Built-In OutputDir.
    Configurez sa valeur par défaut à "C:/StudentFiles/" :

  4. Cliquez sur le bouton [+] (Configure Contexts) dans le coin supérieur droit de la table et cliquez surNew :

  5. Saisissez Production et cliquez sur OK :

  6. Cliquez sur OK à nouveau.
    Configurez la valeur de RepoOutputDir dans la colonne Production à "C:/StudentFiles/Production/" et cliquez sur Finish :

    Rappel : la valeur de chaque variable de contexte comprend un caractère "/" final et est entourée de guillemets doubles.

  7. Le nouveau groupe s'affiche dans le Repository et contient une seule variable :

Ajout des variables de contexte du Repository à un Job

  1. Dans le Job Context_Uppercase, cliquez sur l'ongletContexts.

  2. Cliquez sur le bouton Contexts sous la table Variables. L'assistant Select Context Variables s'ouvre dans une nouvelle fenêtre.

    Cochez la case de la variable précédemment définie afin de spécifier toutes les variables dans ce groupe, puis cliquez sur OK :

    Sélectionnez ici les variables d'un groupe que vous souhaitez rendre disponible dans le Job.

  3. La variable et le groupe s'affichent dans la vue Contexts.

    Les valeurs sont celles définies dans le Repository et vous ne pouvez les modifier à partir d'ici. Vous pouvez les modifier depuis le Repository.

Utilisation des variables de contexte du Repository

  1. Double-cliquez sur le tFileOutputDelimited pour ouvrir sa vue Component.
    Dans le champ File Name, saisissez context.RepoOutputDir + "CappedOut.csv" :

  2. Cliquez sur l'onglet Contexts et sélectionnez Production dans la liste Default context environment :

     

  3. Allez dans la vue Run et exécutez le Job.

    Le fichier créé durant l'exécution du Job s'affiche dans le répertoire C:/StudentFiles/Production car la variable de contexte par défaut est configurée à Production.

  4. Dans le Repository, sous Contexts, double-cliquez sur LocationContextVariables pour ouvrir le groupe et le modifier.
    Cliquez surNext :

  5. Modifiez la valeur dans la colonneProduction en C:/StudentFiles/New_ Production/ et cliquez sur Finish :

  6. Lorsqu'une boîte de dialogue vous propose de propager les modifications, cliquez sur Yes :

    Cette boîte de dialogue vous avertir que la modification va être propagée à tous les Jobs utilisant cette variable du Repository.

  7. La boîte de dialogue Update Detection s'ouvre. Elle liste les éléments d'un Job utilisant cette variable de contexte. Si plusieurs Jobs utilisent cette variable, vous les voyez tous listés, pour pouvoir décider de celui que vous souhaitez modifier.

    Laissez tous les éléments sélectionnés et cliquez surOK :

  8. Exécutez le Job avec le contexte Production à nouveau. Notez que le fichier CappedOut.csv est créé dans le nouveau répertoire, comme attendu en changeant la valeur de la variable de contexte.

    Une fois la variable de contexte configurée dans le Repository, vous pouvez l'utiliser dans n'importe quel Job de votre projet.
