Modification du Job

Vue d'ensemble

Lorsque vous travaillez sur un projet volumineux, vous pouvez sauvegarder la version courante d'un Job avant d'effectuer des modifications, au cas où vous devriez annuler des modifications.

Création d'une nouvelle version

  1. Fermez le Job FilterStates_tMap. Cliquez-droit sur le Job dans le Repository et sélectionnez Open another version :

  2. L'assistant Open another version s'ouvre :

    Dans cette fenêtre, vous pouvez ouvrir une version existante ou en créer une nouvelle.

  3. Cochez la case Create new version and open et cliquez sur le bouton M. Notez que le numéro de version passe de 0.1 à 1.0. Cliquez surFinish :

    Le bouton M incrémente le numéro majeur et le bouton m incrémente le numéro mineur.

  4. Dans la vue Job, vous pouvez voir le nouveau numéro de version :

  5. Remarquez également les numéros de version dans l'onglet Version :

Modification de la nouvelle version du Job

  1. Double-cliquez sur le tMap et modifiez le critère de filtre de la table CappedOut en "AL".equals(row2.StateCode), puis cliquez sur OK :

  2. Double-cliquez sur le composant de sortie CA pour ouvrir sa vue Component et modifiez la valeur du File Name en"C:/StudentFiles/AL_Out.csv" :

  3. Cliquez sur l'onglet View et modifiez la valeur du champ Label format en AL :

  4. Sauvegardez le Job mais ne l'exécutez pas.

Construction et exécution

  1. Construisez à nouveau le Job. Assurez-vous d'utiliser la version 1.0 :

  2. Extrayez le contenu de l'archive, exécutez le script et examinez le nouveau fichier de sortie (si vous avez extrait vers le même dossier, vous devez écraser l'ancien contenu). Le nouveau fichier a été créé, ce qui prouve que le Job a bien été exécuté.
