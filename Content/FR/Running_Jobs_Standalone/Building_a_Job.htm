Construction d'un Job

Vue d'ensemble

Au cours d'une leçon précédente, vous avez exporté des Jobs parents et enfants. Ces Jobs peuvent être partagés avec d'autres intervenants, mais ne s'exécutent que dans le Studio Talend. Dans de nombreuses situations, vous devrez exécuter un Job Talend indépendamment du Studio Talend. Pour ce faire, vous devez construire et exporter le Job.

Construction du Job

  1. Cliquez-droit sur le Job FilterStates_tMap dans le Repository et sélectionnez Build Job :

  2. Cette option vous permet d'exporter le Job dans différents formats avec plusieurs options.
    L'assistant suivant s'ouvre :

  3. Cliquez sur le bouton Browse..., naviguez jusqu'au dossier C:/StudentFiles et laissez Filter_States_tMap.zip dans le champ File name. Cliquez sur Save :

  4. Dans la liste Select the build type, laissez sélectionnée l'option Standalone Job et décochez les cases Context scripts et Java sources.
    Cliquez surFinish :


    L'option Context scripts est utile si vous avez créé des variables de contexte dans votre Job et que vous souhaitez les avoir écrites dans un fichier de configuration.
    L'option Java sources est utile si vous avez besoin des fichiers source Java afin de les recompiler hors du Studio Talend, dans un autre environnement de développement.

Extraction et exécution

  1. À l'aide de l'explorateur Windows, trouvez le fichier que vous venez de créer, FilterStates_tMap.zip et extrayez le contenu.

  2. Trouvez le fichier FilterStates_tMap_run.bat (généralement placé dans un sous-dossier Filter_States_tMap).


    Remarquez qu'un fichier de script shell Linux se trouve dans le même répertoire.

  3. Double-cliquez sur le FilterStates_tMap_run.bat pour l'exécuter.

    Une invite de commande s'ouvre brièvement lorsque le Job s'exécute, mais comme le Job n'a pas de résultat visible, il n'est pas simple de voir ce qu'il s'est passé.
    Cependant, rappelez-vous que ce Job a généré trois fichiers de sortie : CA_Out.csv, AK_Out.csv et NY_Out.csv. Vérifiez que ces fichiers ont bien été créés dans le dossier C:/StudentFiles et que la date et l'heure associées à ces fichiers correspondent à l'heure d'exécution du FilterStates_tMap_run.bat.

Étape suivante
