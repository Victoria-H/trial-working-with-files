Accès à un Service Web

Vue d'ensemble

Le Studio Talend vous permet d'interagir avec un Service Web en tant que partie d'un projet. Dans cet exemple, vous récupérez des informations météorologiques d'un Service Web gratuit pouvant être utilisé pour tester les appels de Services Web.

Notez que ce Service Web est utilisé à des fins de test et peut ne pas donner de résultats pour certaines entrées (comme les codes postaux).

Récupération d'informations du Service Web

  1. Créez un nouveau Job. Cliquez-droit sur Job Designs > Standard et créez un nouveau dossier nommé WS. Cliquez-droit sur le nouveau dossier et sélectionnez Create Standard Job :

  2. Saisissez WeatherWebService dans le champ Name, Get Weather dans le champ Purpose, puis une Description de votre choix.

    Cliquez sur Finish :

  3. Ajoutez un composant tESBConsumer et double-cliquez sur le composant :

  4. Saisissez l'URL suivante dans le champ WSDL :
    http://wsf.cdyne.com/WeatherWS/Weather.asmx?WSDL.
    Appuyez sur la touche Entrée de votre clavier (ou cliquez sur le bouton de rafraîchisssement ) pour alimenter les zones Port Name et Operations.

    5. Configurez le composant comme suit :

    Port Name : WeatherSoap12.

    Operation : GetCityWeatherByZIP(GetCityWeatherByZIP):GetCityWeatherByZIPResponse

    Cochez la case Populate schema to repository on finish puis cliquez sur Finish :

  5. Dans le Repository, naviguez jusqu'à Metadata > File xml >  ws-cdyne-com.
    Ouvrez les sous-dossiers jusqu'à afficher la requête metadata de l'opération SOAP GetCityWeatherByZIP.
    Cette requête doit contenir une seule colonne : ZIP.

     

  6. Saisissez le nom du composant tFixedFlowInput comme entrée de votre Job, placez ce composant à gauche du tESBConsumer :

  7. Placez un tXMLMap entre ces deux composants :

  8. Saisissez le nom du dernier composant tLogRow et placez-le à droite du tESBConsumer :

  9. Cliquez-droit sur le tFixedFlowInput et reliez-le au tXMLMap à l'aide d'un lien Main :

  10. Cliquez-droit sur le tXMLMap, sélectionnez Row > *New Output* (Main) et nommez le lien ZIP.

    Cliquez surOK :

  11. Cliquez sur Yes pour répondre à la question Do you want to get the schema of the target component? :

  12. Cliquez-droit sur le tESBConsumer, sélectionnez Row > Response pour le connecter au composant tLogRow :

  13. Le premier composant affiche un avertissement (! blanc sur fond rouge), indiquant qu'aucun schéma n'a été défini ( No schema has been defined yet) :

  14. Double-cliquez sur le tFixedFlowInput pour ouvrir sa vue Component, cliquez sur le bouton [...] à côté du champ Edit schema.
    Cliquez sur le bouton [+] pour ajouter une colonne, que vous nommez ZIP.

    Cliquez surOK :

  15. Saisissez "94022" dans la colonne Value :

  16. Double-cliquez sur le composant tXMLMap pour ouvrir son éditeur de mapping.
    Dans la table de sortie, cliquez-droit sur payload et sélectionnez Import From Repository :

  17. Naviguez dans le référentiel, sous le dossier Metadata, jusqu'à trouverGetCityWeatherByZIP 0.1.
    Cliquez dessus puis sur OK :

  18. Vous pouvez voir que la métadonnée a été ajoutée au payload dans votre composant tXMLMap :

  19. Mappez les éléments ZIP à l'aide d'un glisser-déposer de l'élément ZIP de gauche, dans la colonne Expression vide à droite, à côté de tns:ZIP.
    Cliquez surOK :

  20. Exécutez le Job et explorez le résultat :

    Notez que le message reçu respecte la métadonnée GetCityWeatherByZIPResponse. Le Service Web public a retourné les informations météorologiques pour Los Altos CA (Californie) (la ville dont le code postal est 94022). Bien entendu, vous pouvez vérifier les informations via un site Web différent, par exemple www.accuweather.com.

Étape suivante

Vous avez terminé la première partie de cet exercice, vous allez à présent modifier le Job pour inclure une sortie plus détaillée.
