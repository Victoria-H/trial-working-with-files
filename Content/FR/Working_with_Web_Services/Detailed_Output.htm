Sortie détaillée

  1. Ajoutez un autre tXMLMap au Job existant et cliquez-droit sur le tLogRow_1 pour le connecter à ce dernier, à l'aide d'un lien Main :

  2. Ajoutez un autre tLogRow et reliez-le au second tXMLMap à l'aide d'un lien *New Output*(Main) :

  3. Nommez la nouvelle sortie Details puis cliquez sur OK :

    Comme précédemment, le second tXMLMap indique une erreur, car les données des colonnes du schéma ne sont pas définies.

  4. Double-cliquez sur le second tXMLMap. Dans la table d'entrée, cliquez-droit sur payload et sélectionnez Import From Repository :

  5. Ouvrez le dossier Metadata et le sous-dossiers suivants pour trouver l'entrée GetCityWeatherByZIPResponse0.1.
    Sélectionnez-la et cliquez sur OK :

  6. Regardez la structure du message de réponse. Sélectionnez la réponse complète.

    Pour ce faire, cliquez sur tns:Success, appuyez sur la touche Maj de votre clavier, maintenez-la enfoncée puis cliquez sur tns:Remarks pour sélectionner tous les éléments de réponse :

  7. Glissez-déposez ces éléments dans la table de sortie Details. Vous allez devoir modifier le nom de chaque colonne, une par une, en supprimant le caractère ":" en début de champ.
    Cliquez sur OK pour chaque élément affiché. Par exemple, la colonne Success :

    Répétez l'action pour les autres colonnes.

  8. Après les modifications, les éléments mappés doivent ressembler à ceci :

    Cliquez sur OK après vérification des noms de colonnes.

  9. Double-cliquez sur le tLogRow_2 puis cliquez sur le bouton Sync columns.

    Afin de vérifier la synchronisation, cliquez sur le bouton [...] devant Sync columns :

     

  10. Le schéma s'affiche. Les colonnes de la table d'entrée doivent être mappées aux colonnes de la table de sortie.

    Vérifiez que les colonnes ont bien été synchronisées, puis cliquez sur OK :

Exécution du Job

  1. Ouvrez l'onglet Run (Job WeatherWebService) et cliquez sur le bouton Run :

  2. Le Job est exécuté et les résultats sont affichés. Cochez/décochez la case Wrap pour voir la différence dans les résultats affichés :

  3. Dans la vue Component du tLogRow_2, sélectionnez Table (print values in cells of a table) dans la zone Mode.

  4. Exécutez à nouveau le Job. Le résultat doit ressembler à ceci :

Désactivation d'éléments

Il est important de savoir que les éléments peuvent être désactivés et activés selon vos besoins.

  1. Désactivez le composant tLogRow_1 dans le Job.
    Cliquez-droit sur le tLogRow_1 et sélectionnezDeactivate tLogRow_1 :

    Le composant devient transparent dans l'espace de modélisation graphique.

  2. Exécutez à nouveau le Job et vérifiez la sortie :

    Notez que la sortie XML du tLogRow_1 n'est plus affichée dans la console.

  3. Décochez la case Wrap et remarquez la modification dans le résultat affiché. Vous devez faire défiler horizontalement pour voir les données de réponse car elles ne sont plus sous l'effet de l'option Wrap :

  4. Cliquez-droit sur le tLogRow_1 et sélectionnez Activate tLogRow_1.

    Si vous exécutez à nouveau le Job, vous verrez à nouveau les composants tLogRow_1 et tLogRow_2 en sortie.
