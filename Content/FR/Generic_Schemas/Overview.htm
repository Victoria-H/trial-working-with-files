Schémas génériques

Vue d'ensemble

Vous allez commencer un nouveau scénario.

Le scénario est le suivant : vous allez prendre des données brutes relatives à des ventes, provenant de différents magasins de détail, effectuer une jointure sur ces données et les informations clients et produits existantes et charger les données combinées dans une table de données de ventes combinées.
Dans ce scénario, les données de ventes n'existent pas encore, vous allez créer un Job pour générer les données. (Une fois les magasins en ligne et les processus en place, un Job ayant le même résultat peut fonctionner avec les vraies données des magasins plutôt qu'avec les données générées.)

Voici un diagramme haut niveau présentant le flux de données du scénario :

 

Au cours de cette leçon, vous allez préparer les schémas, fichiers et métadonnées nécessaires au travail avec les bases de données. Voici les étapes de la leçon :

  • Définir un schéma générique pour les fichiers de données des ventes
  • Générer trois fichiers de données de ventes (un pour chaque magasin)
  • Définir la métadonnée pour le fichier clients (le fichier est fourni)
  • Définir la métadonnée pour le fichiers des produits (le fichier est fourni)

Objectifs

Après avoir terminé cette leçon, vous pourrez :

  • Créer un schéma générique

  • Générer des fichiers contenant des données aléatoires

  • Capturer toutes les informations de vos fichiers dans des variables de contexte
  • Utiliser des sous-jobs

Étape suivante

La première étape consiste à définir un schéma générique pour les données relatives aux ventes et à générer trois fichiers de données de ventes suivant le schéma générique précédemment créé.
