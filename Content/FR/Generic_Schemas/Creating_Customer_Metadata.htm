Création de la métadonnée Customers

Vue d'ensemble

L'étape suivante consiste à créer une métadonnée relative au fichier client afin de l'utiliser pour configurer un composant. Vous allez exporter les informations de métadonnées en tant que variables de contexte, qui est le meilleur moyen de gérer vos variables.

Création d'un fichier de métadonnée

  1. Dans le Repository, sous Metadata, cliquez-droit sur File delimited et sélectionnez Create file delimited.
    Vous allez créer une métadonnée relative au fichier client. Vous avez créé une métadonnée de fichier délimité similaire dans une leçon précédente :

  2. Nommez la métadonnée RetailCustomers, renseignez les champs Purpose et Description, puis cliquez sur Next :

  3. Cliquez sur Browse afin de trouver le fichier C:/StudentFiles/refCustomer.csv, dans la liste Format, sélectionnez WINDOWS, puis cliquez sur Next :

Configuration de la métadonnée de fichier délimité

  1. Dans la zone Rows To Skip, dans le champ Header, saisissez 5, indiquant que les cinq premières lignes du fichier ne contiennent pas de données.
    Dans l'ongletPreview, cochez la case Set heading row as column names, afin de spécifier que la première ligne de données dans le fichier contient les noms de colonnes, puis cliquez sur le bouton Refresh Preview.

    Note : la valeur dans le champ Header a été incrémentée à 6, une fois la case Set heading row as column names cochée.


    Remarquez que la zone Preview affiche le contenu du fichier client dans un format lisible.

  2. Avant de finaliser la création de la métadonnée, exportez toutes les informations en tant que variable de contexte.
    Cliquez sur le bouton Export as context.

  3. Dans l'assistant Create / Reuse a context group, cliquez sur Next.

  4. Laissez RetailCustomers dans le champ Name et saisissez du texte dans les champs Purpose et Description.

  5. Cliquez sur Next et toutes les variables de contexte créées seront affichées :

    Cliquez sur Finish mais ne fermez pas l'assistant principal. Vous allez configurer le schéma Customer dans la section suivante.

  6. Dans votre Repository, sous Contexts, vous pouvez déjà voir votre nouveau contexte RetailCustomers.
    Toutes les informations concernant votre fichier, comme son nom, son séparateur de champs ou son nombre de lignes d'en-tête, sont stockées dans les variables de contexte.

    Il est temps de finaliser la création de la métadonnée de fichier délimité.

Configuration du schéma Customer

  1. Cliquez sur Next dans la page File - Step 3 of 4 de l'assistant.

  2. Modifiez le nom en CustomerMetadata et apportez les modifications suivantes au schéma :

    • Cochez la case Key de la colonne id pour la définir comme colonne clé.

    • Changez la longueur Length de CustomerAddress à 35.

    • Configurez toutes les colonnes en saisissant des noms commençant par Sum (Sum1 à Sum7) de type Float et de longueur Length 11.

    • Configurez la colonne RegTime de type String et de longueur Length 19.

    • Configurez la colonne RegisterTime de type Date et de format "yyyy-MM-dd hh:mm:ss.SSS".

    Vérifiez que votre étape File - Step 4 of 4 correspond à ce qui est affiché ci-dessous et cliquez sur Finish lorsque vous avez terminé :

    Le Studio Talend a extrait le schéma depuis les données et vous avez corrigé le schéma pour qu'il soit plus précis.
