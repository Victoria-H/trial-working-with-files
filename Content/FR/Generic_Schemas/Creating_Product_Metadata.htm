Création de la métadonnée Products

Vue d'ensemble

La prochaine étape consiste à créer une métadonnée pour les fichiers des produits, pour pouvoir les utiliser pour configurer un composant.
Vous allez également exporter les informations de la métadonnée en tant que variables de contexte, ce qui représente le meilleur moyen de gérer vos variables.

Création d'une métadonnée de fichier délimité

  1. Créez une nouvelle métadonnée de fichier délimité et nommez-la RetailProducts :

  2. Saisissez RetailProducts dans le champ Name et renseignez les champs Purpose et Description.
    Cliquez sur Next :

  3. Configurez la métadonnée avec les informations suivantes :

    Le fichier est C:/StudentFiles/refProducts.csv et le Format est Windows :

  4. Il n'y a aucune ligne d'en-tête à ignorer. Ne cochez pas la case Set heading row as column names (aucune configuration nécessaire dans l'étape 3).
    Cliquez sur le bouton Export as context :

  5. Dans l'assistant Create/Reuse a context group, cliquez sur Next.

  6. Dans le champ Name, laissez RetailProducts et renseignez les champs Purpose et Description, puis cliquez sur Next :

  7. Les variables de contexte et les valeurs sont affichées :

    Cliquez sur Finish mais ne fermez pas l'assistant principal.
    Vous devez finaliser la création de la métadonnée de fichier délimité, en définissant le schéma Products dans la section suivante.

Configuration du schéma Products

  1. Cliquez sur Next dans l'étape File - Step 3 of 4.

  2. Dans le champ Name, saisissez ProductMetadata.
    Saisissez id dans la première colonne et définissez son Type à Integer et sa longueur Length à 3.
    Saisissez Product dans la seconde colonne, laissez son Type à String et sa longueur Length à 10.
    Cliquez sur Finish :

  3. Vérifiez que la métadonnée a bien été créée dans le Repository :
