Création d'un schéma générique et génération de fichiers de données

Vue d'ensemble

Au cours de cette leçon, vous avez créé un schéma générique dans le Repository. Ce schéma va décrire les informations sur les données des ventes. Le schéma générique va être utilisé pour générer des fichiers de données de ventes.

Création du schéma générique

  1. Dans la vue Repository, sous le dossier Metadata, cliquez-droit sur Generic schemas et sélectionnez Create generic schema :

  2. Au cours de la première étape, saisissez ShopMetadata dans le champ Name.
    Dans le champ Purpose, saisissez : sales data.
    Saisissez Sales data for database lab dans le champ Description.
    Cliquez surNext :

  3. Dans la seconde étape, dans le champ Name, saisissez Shopmetadata.
    Cliquez quatre fois sur le bouton [+] pour ajouter quatre colonnes à la table.
    Nommez la première colonne Shopname, sélectionnez String dans la colonne Type et saisissez 10 dans la colonne Length.
    Pour les colonnes suivantes, nommez-les CustID, ProductID et Quantity et sélectionnez Integer dans la colonne Type.

  4. Cliquez sur Finish. Le schéma final dans le Repository doit ressembler à ceci :

Génération des fichiers de données de ventes

  1. Cliquez-droit sur Standard sous Job Designs, dans le Repository et sélectionnez Create folder.

  2. Dans le champ Label, saisissez Sales et cliquez sur Finish :


  3. Cliquez-droit sur le dossier Sales sous Job Designs / Standard et sélectionnez Create Standard Job :

  4. Saisissez SalesFilesPrep dans le champ Name.

    Dans le champ Purpose, saisissez create sales data, le texte de votre choix dans le champ Description puis cliquez surFinish :

  5. Le nouveau Job créé s'ouvre dans l'espace de modélisation graphique.
    Saisissez le nom du tRowGenerator pour ajouter ce composant au Job :

  6. Ouvrez sa vue Component et sélectionnez Repository dans la liste Schema.
    Cliquez sur le bouton [...] à côté du champ Schema :

  7. Ouvrez les dossiers Generic schemas et ShopMetadata et sélectionnez Shopmetadata.
    Cliquez surOK :

  1. Double-cliquez sur le tRowGenerator. Le schéma doit vous sembler familier. Saisissez 5000 dans le champ Number of Rows for RowGenerator afin de créer 5000 entrées.

  2. Sélectionnez la colonne ShopName et saisissez "Shop1" dans la colonne Value de la table de la zone inférieure.

  3. Sélectionnez la colonne CustID et sélectionnez Numeric.random dans la colonneFunctions.

    Dans l'onglet Function parameters, saisissez 1 pour min value et laissez 100 pour max value :

    Cela génère un nombre aléatoire entre 1 et 100 pour la colonneCustID.

  4. Configurez la colonne ProductID pour générer un nombre aléatoire entre 1 et 80, puis la colonne Quantity pour générer une valeur entre 1 et 1000 et vérifiez les paramètres.
    Cliquez surOK :

  5. Saisissez le nom du composant tFileOutputDelimited dans l'espace de modélisation graphique :

  6. Cliquez-droit sur le tRowGenerator et sélectionnez Row >Main pour le connecter au composant de sortie :

  7. Double-cliquez sur le composant tFileOutputDelimited pour ouvrir sa vue Component.

    Cliquez sur le bouton [...] à côté du champ File Name et configurez la valeur à "C:/StudentFiles/SalesFiles/shop1.csv".
    Laissez la valeur du champ Field Separator à celle par défaut : ";".

    Note : assurez-vous de placer le fichier de sortie dans le sous-dossierSalesFiles.

  8. Dans la vue Component, cliquez sur l'onglet View et saisissez Shop1out dans le champLabel format :

  9. Cliquez dans le cadre du sous-job, dans l'espace de modélisation graphique.
    Cliquez sur l'onglet Component pour ouvrir les paramètres du sous-job :


  10. Cochez la case Show subjob title dans l'onglet Basic settings.
    Dans le champ Title, saisissez Shop1 :

    Le titre du sous-job s'affiche en haut du sous-job dans l'espace de modélisation graphique.

  11. Sauvegardez votre Job en appuyant sur les touches Ctrl+S.
