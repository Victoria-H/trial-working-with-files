Capture des rejets de jointure

Configuration du modèle de jointure dans le tMap

  1. Double-cliquez sur le composant tMap. Dans la table row2, cliquez sur le bouton tMap settings (représentant une clé anglaise) :

    Vous pouvez configurer des paramètres supplémentaires relatifs à la table. Notez que la valeur sélectionnée pour Join Model est Left Outer Join.
    Une jointure Left Outer Join prend en compte toutes les lignes de la table principale, même s’il n’y a pas de ligne dans la table de référence avec une valeur correspondante dans la colonne de jointure, ce qui explique pourquoi certaines lignes n’ont pas pris en compte les noms d’États. Mais vous avez besoin de savoir pourquoi certaines lignes de référence ne correspondent pas. Pour cela, vous devez utiliser une jointure Inner Join.
    Cliquez sur Left Outer Join dans la colonne Value, de la ligne Join Model, puis cliquez sur le bouton [...] pour modifier la valeur :

  2. La boîte de dialogue Options s'ouvre. Sélectionnez Inner Join puis cliquez sur OK. En changeant le type de jointure à Inner Join, les lignes ne correspondant pas seront exclues de la sortie et vous pourrez capturer les rejets à des fins de débogage :

Ajout de la table des rejets

  1. Toujours dans le tMap, cliquez sur le bouton [+] (Add output table) au-dessus de la table CappedOut :

    Vous créez une nouvelle table afin de capturer les lignes rejetées par la jointure Inner Join, afin de déterminer où se situe le problème.

  2. Saisissez JoinFailures dans le champ de texte pour nommer la nouvelle sortie, puis cliquez sur OK :

  3. La nouvelle table s'affiche sous CappedOut :

  4. Cliquez sur le bouton tMap settings (représentant une clé anglaise) dans la table JoinFailures :

    Notez que les paramètres d'une table de sortie sont différents de ceux d'une table d'entrée.

  5. Modifiez la valeur du champ Catch lookup inner join reject et passez-la à true. Cliquez sur OK :

  6. Déposez Last et State de la table row1 dans la table JoinFailures :

    Notez que cette sortie va contenir le nom de famille et l'État des lignes rejetées par la jointure Inner Join.

  7. Cliquez sur Ok.

Affichage de la sortie

  1. Ajoutez un composant tLogRow sous le tMap.

    Souvenez-vous, ce composant écrit des lignes dans la console de la vue Run :

  2. Cliquez-droit sur le composant tMap, sélectionnez Row > JoinFailures, puis cliquez sur le tLogRow pour relier les composants :

  3. Le lien JoinFailures correspond à la table de sortie ajoutée au composant tMap :

Exécution du Job

  1. Exécutez le Job et examinez la sortie dans la console :

    Notez que les codes d'États dans les lignes rejetées sont en minuscules (du fichier d'entrée), ce qui signifient qu'ils ne correspondent pas aux codes en majuscules du fichier de référence.

Étape suivante

Maintenant que vous avez identifié le problème, vous pouvez le corriger, pour que tout fonctionne comme prévu.
