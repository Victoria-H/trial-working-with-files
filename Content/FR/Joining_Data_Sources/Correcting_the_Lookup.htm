Correction du flux Lookup

Vue d'ensemble

Vous savez que la jointure échoue à cause des minuscules dans les chaînes de caractères, vous pouvez donc corriger la configuration.

Configuration du tMap

  1. Double-cliquez sur le composant tMap. Rappelez-vous que row2 est la table de référence pour la jointure. Dans la table row2 à gauche, cliquez surrow1.State puis sur le bouton [...] pour ouvrir l' Expression Builder :

  2. Dans un exercice précédent, vous avez utilisé l' Expression Builder pour convertir des chaînes de caractères minuscules en lettres capitales. Vous devez faire la même chose pour cette colonne, mais d'une autre manière. Après le dernier caractère de row1.State, saisissez directement .to et vous verrez les options proposées par l'autocomplétion contextuelle de l'éditeur.

    Double-cliquez sur toUppercase(): String - String pour la sélectionner.

  3. Cliquez sur Ok pour fermer l' Expression Builder. La valeur de la colonne State est convertie en lettres capitales avant comparaison à la table de référence, pour que la jointure fonctionne correctement :

  4. Cliquez sur Ok pour fermer l'éditeur du tMap.

Exécution du Job

  1. Exécutez le Job. Notez que le tLogRow n'a traité aucune ligne, car aucun rejet n'a été trouvé. Le flux de données en sortie est de 100 lignes traitées.

  2. Examinez le contenu du fichier de sortie avec l'option Data Preview dans le Studio ou avec Notepad++, pour vous assurer que toutes les lignes comprennent le nom de l'État :


    Vous n'avez plus besoin d'afficher les rejets, vous pouvez nettoyer le Job en supprimant le tLogRow.

Nettoyage du Job

  1. Cliquez-droit sur le composant tLogRow dans l'espace de modélisation graphique et sélectionnez Delete pour supprimer le composant :

    Notez que vous pouvez également supprimer un composant en utilisant la touche Suppr de votre clavier.

  2. Double-cliquez sur le tMap pour ouvrir son éditeur de mapping.
    Cliquez sur la table JoinFailures à droite puis cliquez sur le bouton Remove selected output table, marqué d'une croix rouge (en haut de la fenêtre).

    Cliquez sur OK lorsqu'une boîte de dialogue vous demande de confirmer la suppression de la table JoinFailures :

    Cliquez sur Ok pour fermer l'éditeur de mapping

  3. Cliquez sur Clear dans la vue Run afin de supprimer l'affichage de statistiques dans l'espace de modélisation graphique :

  4. Sauvegardez le Job.

Étape suivante

Vous avez terminé la leçon relative à ce Job, vous pouvez passer à la Conclusion.
