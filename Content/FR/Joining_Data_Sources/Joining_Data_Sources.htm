Création d'une jointure

Vue d'ensemble

Au cours de cette section, vous allez dupliquer le Job créé dans la leçon précédente.

Vous allez ensuite utiliser le fichier contenant les noms et codes des États dans une table de référence (lookup) pour inclure le nom complet de l'État dans la sortie.

Duplication du Job

  1. Dans le Repository, sous Job Designs, cliquez-droit sur UpperCase et sélectionnez Duplicate :

  2. Saisissez LookupState dans le champ Input new name, puis cliquez sur OK :

  3. Double-cliquez sur LookupState dans le Repository pour l'ouvrir :

    Prenez un moment pour examiner les composants et vous souvenir de la fonction du Job.

Ajout d'une seconde source

  1. Glissez le nouvel élément StateCodes du noeud Metadata du Repository dans l'espace de modélisation graphique, au-dessus du tMap.
    Astuce     : Si vous n'avez pas de place, cliquez sur le groupe de composants et déposez-le plus bas dans l'espace de modélisation graphique.

  2. Cette fenêtre vous permet de choisir le composant à utiliser avec la métadonnée.
    Sélectionnez le tFileInputDelimited et cliquez surOK :

  3. Votre Job doit ressembler à ceci :

    Notez que le composant est nommé de la même manière que l'élément de métadonnée. Lorsque vous stockez des informations de configuration en tant que métadonnées dans le Repository, vous pouvez utiliser cette métadonnée comme point de départ, comme ici, plutôt que de placer un composant et de le configurer ensuite.

  4. Cliquez-droit sur StateCodes, sélectionnez Row > Main et cliquez sur le composant tMap pour créer le lien :

    Notez que le lien est un lien Lookup et non Main. Vous pouvez fournir plusieurs sources en entrée, pour un composant tMap, mais une seule via un lien Main. Les autres sources seront liées par des liensLookup.

  5. Double-cliquez sur le StateCodes pour ouvrir sa vue Component :

    Notez que la valeur des listes Property Type et Schema est Repository et non Built-In.

  6. Double-cliquez sur le composant CustomersFile pour voir que ce composant utilise des informations de configuration Built-In.
    Le type de propriété Built-in est spécifique à un composant, alors que le type Repository utilise des informations stockées en tant que métadonnées et pouvant être utilisées par différents composants dans différents Jobs.

Configuration du tMap

  1. Double-cliquez sur le tMap :

    Notez qu'il y a une seconde table d'entrée sur la gauche, nommée row2, correspondant au lien d'entrée reliant le StateCodes au tMap.

  2. Déposez State de la table row1 dans le champ Expr. key de la colonne StateCode de la table row2 :

  3. Cela crée la jointure, la valeur de la colonne State de la table row1 sera comparée à la valeur de la colonne StateCode de la table row2 :

  4. Déposez StateName de la table row2 au bas de la table CappedOut, à droite :

    Cela ajoute la valeur de la colonne StateName à la sortie.

  5. Cliquez sur Ok.

  6. Le système vous propose de propager vos modifications. Cliquez sur Yes :

  7. Sauvegardez le Job.

Exécution du Job

  1. Exécutez le Job :

  2. Examinez le fichier de sortie.

    Cliquez-droit sur le tFileOutputDelimited et sélectionnez Data viewer.
    Notez que toutes les lignes ne comprennent pas le nom de l'État :

    Cela signifie que le Job ne fait pas exactement ce que vous souhaitez, vous devez comprendre pourquoi.

  3. Cliquez sur Clear pour effacer l'affichage des statistiques dans l'espace de modélisation graphique et la console :

Étape suivante

L'étape suivante consiste à déterminer pourquoi le Job n'effectue pas correctement sa tâche, afin de le corriger.
