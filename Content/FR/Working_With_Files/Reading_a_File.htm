Lecture d'un fichier d'entrée

Vue d'ensemble

Au cours de la première leçon, vous avez créé un premier Job simple. Dans cette section, vous allez créer un nouveau Job lisant des données d'un fichier .csv délimité contenant des données clients. Le Job sera créé dans un dossier spécifique qui contiendra tous les Jobs relatifs au scénario concernant les données clients.

Création d'un nouveau Job

  1. Créez un nouveau dossier pour vos Jobs. Sous Job Designs, cliquez sur le [+] devant Job Designs, cliquez-droit sur Standard et sélectionnez Create folder :

    Dans le champ Label, saisissez Customers puis cliquez sur Finish :

    Le nouveau dossier s'affiche sous Job Designs > Standard :

  2. Pour créer le Job suivant, dans le Repository, cliquez-droit sur Customers, sous Job Designs > Standard, puis cliquez sur Create Standard Job :

  3. Saisissez UpperCase dans le champ Name et renseignez les champs Purpose et Description comme suit, puis cliquez surFinish :

    Rappelez-vous que ce Job va convertir le texte de la colonne State en majuscules. Seul le nom est requis, mais la bonne pratique recommande de renseigner les autres champs pour documenter le Job.

  4. Le nouveau Job s'ouvre, prêt à être construit :

Ajout d'un composant de lecture de fichier

  1. Dans la Palette, cliquez sur File, puis sur Input et faites défiler vers le bas, jusqu'à trouver le composanttFileInputDelimited :

    Les composants dans la Palette sont groupés par fonction. Ce composant particulier lit des données d'un fichier délimité.

  2. Cliquez sur le tFileInputDelimited dans la Palette, puis cliquez dans l'espace de modélisation graphique pour placer le composant.

    Notez le triangle jaune avec un point d'exclamation (!) dans le coin supérieur droit du composant, indiquant un avertissement.

  3. Placez votre curseur sur l'icône pour lire l'avertissement :

    L'avertissement spécifie que le composant nécessite un schéma et doit être relié à un composant de sortie. Vous fournirez le schéma et le lien lors de la construction du Job.

Configuration

  1. Double-cliquez sur le tFileInputDelimited pour ouvrir sa vue Component sous l'espace de modélisation graphique.
    Cliquez sur le bouton [...] à côté du champ File Name / Stream, pour parcourir votre système jusqu'au fichier d'entrée :

  2. Trouvez le fichier dans C:/StudentFiles/Custs.csv et cliquez sur Open :

  3. Dans la vue Component, dans le champ Field Separator, saisissez "," pour indiquer qu'une virgule sépare les valeurs des colonnes dans le fichier et non le point-virgule par défaut. Modifiez la valeur dans le champ Header à 1 afin de spécifier que la première ligne du fichier d'entrée contient le nom des colonnes :

    4. Note : vous pouvez spécifier ici toutes les informations nécessaires pour que le composant effectue ses tâches. Notez que les onglets à gauche, comme Advanced settings et View, vous permettent d'accéder à des paramètres supplémentaires de configuration.

  4. À l'aide de l'Explorateur Windows, trouvez le fichier Custs.csv, cliquez-droit dessus et sélectionnez Edit with Notepad++ :

  5. Explorez le contenu du fichier :

    Notez que le séparateur dans le fichier est la virgule, comme précédemment configuré et que la première ligne contient l'en-tête. Dans l'en-tête, vous pouvez voir la liste des champs que vous allez définir comme schéma de votre fichier dans les étapes suivantes.

Schéma

  1. Dans la vue Component, cliquez sur le bouton [...] à côté du champ Edit Schema. La fenêtre Schema s'ouvre :

    Note : vous pouvez spécifier ici le format des données du fichier Custs.csv. Les informations concernant la structure de ce fichier vous sont fournies dans cette leçon. Dans vos propres projets, vous devez connaître le schéma de vos données dans les fichiers source.

  2. Cliquez sur le bouton [+] pour ajouter une colonne au schéma.
    Remplacez newColumn par First et saisissez 15 dans la colonne Length :

    Note : cela spécifie que la première colonne de chaque ligne de données dans le fichier Custs.csv contient une chaîne de 15 caractères, correspondant au prénom du client.

  3. Ajoutez cinq colonnes de type String, renommez-les comme suit et, dans la colonne Length, saisissez respectivement :

    • Last, 15

    • Number, 10

    • Street, 20

    • City, 20

    • State, 2

    Lorsque vous avez terminé, votre schéma doit ressembler à ceci :

    Cliquez surOK.

    Comme vous pouvez voir dans ce schéma, le fichier Custs.csv contient des informations simples d'identification et d'adresses pour un groupe fictif de clients, organisé en six colonnes.

Affichage

  1. Cliquez sur l'onglet View, à gauche, dans la vue Component et saisissez Customers File dans le champ Label format :

    Note : l'ensemble des paramètres de l'onglet View affecte la manière dont s'affiche le composant dans l'espace de modélisation graphique. Le texte saisi ici devient le nom du composant dans l'espace de modélisation graphique. Vous pouvez saisir des balises HTML ou du texte dans les champs, afin de changer le style du libellé et des astuces.

Avertissements

  1. Dans le menu Window, cliquez sur Show View :

    2. Dans cette fenêtre, vous pouvez choisir quelles vues afficher dans l'application.

  2. Développez Talend et sélectionnez Problems. Cliquez surOK :

  3. La vue Problems s'affiche sous l'espace de modélisation graphique.

    Cliquez sur la flèche sur la ligne Warnings afin de la développer :

    4. Note : dans la vue Problems, vous pouvez voir des informations concernant les problèmes relatifs au Job ou aux composants individuels. Dans ce cas, l'avertissement vous indique que votre composant d'entrée doit être relié à une sortie. L'un des avertissements précédents n'est plus affiché car vous avez défini un schéma pour le composant.

Étape suivante
