Transformation de données

Vue d'ensemble

Votre Job UpperCase contient un composant fournissant les données d'entrée. Vous devez ajouter un composant effectuant une transformation : convertir des abréviations d'États de minuscules à majuscules.

Ajout de composants

  1. Saisissez tMap dans le champ de recherche en haut de la Palette puis cliquez sur l'icône rerésentant une loupe, ou appuyez sur la touche Entrée :

    Note : vous pouvez utiliser la fonction de recherche de la Palette pour trouver rapidement un composant, même si vous ne vous souvenez que d'une partie de son nom.

  2. Placez le tMap dans l'espace de modélisation graphique, à droite du composant existant. Cliquez-droit sur le composant Customers File, sélectionnez Row > Main et cliquez sur le tMap. Cela crée un lien entre ces composants :

    Le tMap permet de mapper des données de l'entrée à la sortie et peut effectuer une grande variété de transformations sur les données. Vous allez rapidement l'utiliser comme pièce centrale de nombreux Jobs dans Talend Data Integration.

    Notez l'avertissement affiché pour le composant tMap. Le tMap transforme les données et nécessite une sortie.

Ajout d'un composant de sortie

  1. Cliquez-droit sur le tMap, maintenez le clic et déplacez votre curseur hors du composant.

    Une liste de composants est alors suggérée.

    Saisissez les premiers caractères du nom tFileOutputDelimited dans le champ de recherche, puis double-cliquez sur le composant pour l'ajouter à votre Job :

    Comme son nom l'indique, le tFileOutputDelimited écrit des lignes dans un fichier de format délimité.

  2. Une boîte de dialogue vous demande de saisir un nom pour le lien de sortie. Saisissez CappedOut (les espaces ne sont pas autorisés) dans le champ, puis cliquez sur OK :

  3. Le nouveau lien de sortie est un lien Main, comme le lien d'entrée.

    Passez votre curseur sur l'erreur du tMap ou ouvrez la vue Problems pour voir quelle est l'erreur. Il y a un problème avec le schéma de sortie, qui sera résolue par la suite.

Mapping de lignes

  1. Double-cliquez sur le composant tMap pour ouvrir la fenêtre de configuration :

    Comme le tMap fournit de nombreuses fonctionnalités, la fenêtre comporte de nombreux éléments. Remarquez la table à gauche, nommée row1 et la table à droite, nommée CappedOut. Ces tables représentent le schéma des liens d'entrée et de sortie, dont les noms correspondent aux liens. Le schéma de row1 est copié du composant d'entrée et semble familier. Le schéma de CappedOut n'est pas encore défini.

    L'objectif de ce Job est de passer toutes les données de row1 à CappedOut, en mettant en majuscules les abréviations des États.

  2. Dans row1, cliquez sur la première ligne, appuyez sur la touche Maj, puis cliquez sur la dernière ligne, afin de les sélectionner toutes.

  3. Maintenez la touche Maj enfoncée et déposez toutes les lignes dans la table CappedOut :

    Vous pouvez voir des flèches indiquant le mapping des colonnes d'un schéma à l'autre. Dans la table CappedOut, remarquez la colonne Expression, qui définit le contenu des données et Column, qui définit le nom des colonnes. Les données et les noms de colonnes sont une correspondance exacte par rapport à celles de row1 (l'expression row1.<name> spécifie les données dans la colonne Name de row1).

Construction de l'expression

  1. Comme vous souhaitez modifier les données de la colonne State, cliquez sur row1.State dans la table de sortie CappedOut, puis cliquez sur le bouton [...] qui s'affiche.

  2. Dans la liste Categories, sélectionnez StringHandling, puis faites défiler la liste Functions jusqu'à trouver et sélectionner UPCASE :

    Note : l'Expression Builder vous permet de construire des expressions avec la syntaxe Java. Vous pouvez saisir l'expression en tant que texte vous-même, mais si vous n'êtes pas totalement familier avec la syntaxe, cet outil simplifie le processus. L'expression est toujours row1.State. Cette zone de l' Expression Builder fournit un accès organisé à de nombreuses fonctions. Notez la documentation fournie pour chaque fonction dans la zone Help.

  3. Double-cliquez sur UPCASE pour insérer cette syntaxe dans la zone Expression au point d'insertion (le vôtre peut être au début de l'expression plutôt qu'à la fin) :

    Notez que l'expression se compose de la catégorie, d'un point et du nom de la fonction, avec les arguments entourés de parenthèses. Cette fonction convertit la chaîne de caractères entre les parenthèses en majuscules. Dans l'exemple par défaut, la fonction contient la chaîne de caractères littérale "Hello". Vous ne souhaitez pas convertir de chaîne de caractères littérale mais le contenu de la colonneState.

  4. Déplacez row1.State dans l'expression, pour qu'elle remplace "Hello", puis cliquez surOk :

    Note : l'expression agit sur le bon argument. Pensez à supprimer les guillemets dans les parenthèses.

  5. La nouvelle expression s'affiche dans la table CappedOut :

    6. La colonne de sortie State va contenir la chaîne de caractères de la colonne du même nom de la table row1, convertie en majuscules.

Sauvegarde

  1. Vous avez terminé la configuration du tMap, cliquez sur Ok. Une boîte de dialogue s'ouvre :

    Vous avez modifié le schéma de sortie, cette boîte de dialogue vous propose de propager ces modifications au schéma du composant à l'autre bout du lien.

  2. Cliquez sur Yes.

Configuration de la sortie

  1. Double-cliquez sur le composant de sortie tFileOutputDelimited pour ouvrir sa vueComponent.

  2. Modifiez le nom du fichier de sortie à C:/StudentFiles/CappedOut.csv, dans le champ Field Separator, saisissez une virgule entre guillemets et cochez la case Include Header pour que la première ligne du fichier comprenne les noms de colonnes :

  3. Cliquez sur le bouton [...] à côté de Edit schema :

    Notez que le schéma du composant correspond au schéma spécifié dans le tMap, le schéma d'entrée ( CappedOut).

    Vous n'avez aucune modification à effectuer, cliquez surOK.

Étape suivante

Votre Job est terminé et vous pouvez l'exécuter.
