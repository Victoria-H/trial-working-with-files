Filtre sur les données de sortie

Vue d'ensemble

Votre Job doit filtrer la sortie sur la colonne State code pour que les fichiers de sortie soient séparés par État.

Duplication du Job

  1. Cliquez-droit sur le Job LookupState et sélectionnez Duplicate :

  2. Saisissez Filter_States dans le champ Input new name, puis cliquez sur OK :

  3. Double-cliquez sur le nouveau Job pour l'ouvrir dans l'espace de modélisation graphique.

Ajout d'un filtre

  1. Double-cliquez sur le tMap.

    Cliquez sur l'icône Activate/ unactivate expression filter en haut à droite de la table CappedOut et saisissez "CA".equals(row1.State) comme définition de filtre et cliquez sur Ok :

    Une fois l'expression de filtre correctement définie, une flèche orange est ajoutée, mappant le flux de la table d'entrée à la table de sortie.

  2. Exécutez le Job.

    Notez que seules 9 lignes sont écrites dans le fichier de sortie.

  3. Cliquez-droit sur le composant de sortie tFileOutputDelimited et sélectionnez Data viewer.

    Notez que les données filtrées sont seulement des enregistrements ayant la valeur CA (en majuscules) dans la colonne State du fichier d'entrée Custs.csv.

    Cliquez sur Close.

  4. Double-cliquez à nouveau sur le tMap et modifiez le filtre à "CA".equals(row2.StateCode), puis cliquez sur Ok :

    Une fois de plus, une flèche orange est placée pour vous et identifie le flux de données pour l'expression modifiée. Rappelez-vous que row2.StateCode contient le code d'État en lettres capitales.

  5. Exécutez le Job à nouveau. Vous constatez que la sortie contient 27 lignes :

  6. Cliquez-droit sur le composant de sortie tFileOutputDelimited et sélectionnez Data viewer.

Étape suivante

Vous avez filtré les données pour la colonne State Code, vous allez apprendre à créer plusieurs sorties et à filtrer les différents États dans le composant tMap.
