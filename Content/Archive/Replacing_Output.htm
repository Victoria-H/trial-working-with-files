Replicating Output

Overview

To make the three-way filtering of the purchase data more obvious, you will now use a component to replicate a data stream to multiple outputs.

Create a New Job

 

Show Me!

  1. Duplicate the Job SalesReport, calling the new Job "SalesBySKU".

  2. Close all other Jobs and then open the new one.

  3. Delete the tLogRow component.

  4. Open the second tMap component and remove the constraint filter from the CustomerPurchases table (simply delete the expression text from the box).

  5. Remove the row between the tMap component and the tFilterRow component:

Add a Replicate Component

 

Show Me!

  1. Add a tReplicate component to the design workspace to the right of the tMap component, being careful not to place it on top of an existing row.

  2. Right-click tMap, click Row followed by CustomerPurchases, and then click tReplicate:

  3. Move the tFilterRow component to the right of the tReplicate component, and then remove the W456 row:

  4. Add two more tFilterRow components, for a total of three, and then add one tFileOutputDelimited component, again for a total of three:

  5. Connect the tReplicate component to each tFilterRow component with a Main row (if prompted to propagate changes, click Yes):

  6. Configure each tFilterRow component to filter based on one of the values for the SKU column ("W456", "G23", and "I14"). Note that one component is already configured to filter for "I14".

  7. Connect each tFilterRow component to a tFileOutputDelimited component with a Filter row:

  8. For each of the output components, specify a file name that reflects the SKU value for that component, set the field delimiter to a comma, and then click the Sync Columns button to make sure the schemas are all correct.

  9. Name the rows leading to the output components and the components themselves according to the SKU values:

    The Job now has three duplicate rows coming from the tMap component, each to a component that filters for a specific SKU value and writes the results to a separate file.

Run the Job

Show Me!

Run the Job and then examine the contents of the three different output files. The results are the same as the previous Job, but the design makes the purpose much easier to see.
