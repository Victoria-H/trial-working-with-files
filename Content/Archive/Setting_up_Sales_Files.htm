Setting up Sales Files

Overview

You are still setting up the data required by the primary Job. You have tables of customer and product information, now you need sales data.

Create the Job

  1. Create a generic schema named "ShopSchema" in the Repository for sales data with four columns: ShopName, type String, length 10; CustID, ProductID, and Quantity, all type Integer; and name the metadata "ShopMetadata":

  2. Create a new Job, naming it "SalesFilesPrep" or something similar.

  3. Add a tRowGenerator component to the design workspace and then set the schema to the ShopSchemaRepository schema you just created:

  4. Double-click the tRowGenerator component and then configure it as follows:

    • Create 5000 lines

    • Use a custom function (the default setting in the Functions column, labeled "...") to set the value of ShopName to "Shop1"

    • Generate a random number from 1 to 100 for CustID, from 1 to 80 for ProductID, and from 1 to 1000 for Quantity(click image to enlarge):

  5. Connect the tRowGenerator component to a tFileOutputDelimited component that saves the data in a delimited file named "Shop1" under the directory C:/StudentFiles/Sales Files/:

    Be sure to put the output file under the subdirectory Sales Files.

  6. Click on the subjob and then, click Show subjob title in the Basic settings tab.

  7. In the Title box, enter "Shop 1":

Run Job

  1. Copy and paste the subjob twice.

  2. Change each instance of "Shop1" to "Shop2" in the component configurations in the second subjob, and to "Shop3" in the third. Be sure to change both the value of the ShopName column in the tRowGenerator component and the name of the output file in the tFileOutputDelimited component, along with the names of the components:

  3. Run the Job and then verify the content and location of the output files.
