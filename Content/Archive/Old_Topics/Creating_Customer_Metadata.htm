Create Customer & Product Metadata

Overview

Your next step is to create Metadata about the customer file so that you can use it to configure a component. You will also export Metadata information as context variables which is the best way to manage your variables.

Create Customer File Metadata

  1. In the Repository, under Metadata, right-click File delimitedand then click Create file delimited. You are creating metadata about the customer file. You already created similar file delimited metadata in an earlier training lesson:

  2. Name the metadata RetailCustomers, provide a purpose and description and click Next:

  3. Click Browse to locate the file C:/StudentFiles/refCustomer.csv, change the Format to WINDOWS, and then click Next:

Configure File Delimited Metadata

  1. In the Rows To Skip section, set Header to 6, indicating that the first six rows of the file do not contain data.
    Under Preview, select Set heading row as column names to specify that the first row of data in the file are actually the column names and click Refresh Preview.

    Note: Now the Preview area shows the contents of the customer data file in a reasonable format. Before finalizing your metadata creation, you will export all the information as context variables.

  2. Click Export as context.

  3. On the Create/Reuse a context group wizard, click Next.

  4. Leave RetailCustomers for the Name and enter text for the Purpose and the Description.

  5. Click Next to visualize context variables that will be created and then click Finish:

  6. In your Repository, below Contexts, you will see your new RetailCustomers context. All information about your file, such as the file name, field separator or the number of header lines, are now stored in context variables.

    Now, it's time to finalize the File Delimited metadata creation.

Configure Customer Schema

  1. Click Next in File - Step 3 of 4

  2. Change the name to CustomerMetadata and then make the following changes to the schema:

    • Specify the column id as the key.

    • Change the length of CustomerAddress to 35.

    • Configure all the columns with names that begin with Sum as type Float with a Length of 11.

    • Set the RegTime column as type String with a Length of 19.

    • Set the column RegisterTime as type Date with the format "yyyy-mm-dd hh:mm:ss.SSS"

    Then click Finish:

    Talend Studio derived the schema from the data, and you have now corrected the schema to be accurate.

Create Job

  1. Create a new Job, naming it CustPrep.

  2. Drag the RetailCustomersmetadata onto the design workspace, choosing tFileInputDelimitedas the component. This file input component reads the customer data file. As you exported your information as context variables, you will see the following window that appear. Click Yes:

  3. Double-click RetailCustomers to open the Componentview. The information displayed is just what you entered in the Repositorymetadata and saved as context variables:

    Because you will use the data in this file to load the database table, you know that the schema of the database table needs to match. Rather than entering that schema information again when you configure the database, you can save the schema information separately in the Repositoryto use it again.

 

Create Schema Metadata

  1. Click the ...Icon next to Edit schema:

  2. Click OK:

    Note: Because the schema is defined in the Repositoryand not locally, you are given several choices of how to work with it. In this case, you want to examine and save the schema, not change it.

  3. In the Schema of RetailCustomerswindow, click the Save as generic schemabutton, shaped like a floppy disk:

    Note: This is the schema you defined when you configured the metadata file.

  4. Click OKto leave the schema in the default folder:

    Note: You will be working with the customer data in several components, so saving this information as a generic schema makes it easier to reference in multiple components.

  5. Enter RetailCustomersSchema for the Nameand enter the Purposeand the Description.

    Then clickFinish:

    Click OK to close the Schema of RetailCustomers window.

    Note: Now the schema appears under Metadatain the Repository, under Generic schemas.

  6. Expand the Generic schema under Metadata in the Repository until you see the word metadata. Double click on metadata, change Name to RetailCustomers and then click Finish:
