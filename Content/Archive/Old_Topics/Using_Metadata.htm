Using Metadata

Overview

Now, you will create a new Job and use the Metadata StatesCodesfor creating the input component of your Job. You will start to use the components out of the Processing folder in the Palette starting with the tFileOutputXMLcomponent and you will create another Metadata with the Db Connectionswizard which will be used in the following labs. Finally you will use the tMapcomponent, another very powerful component, to create a typical ETL Job.

Create New Job

  1. Right-click on Job Designs and click Create Standard Job:

  2. Enter UseMetadata for the name and enter your text for the Purpose and the Description and then click Finish:

  3. Expand the folder Metadata in the repository and drag the StatesCodesmetadata to the workspace of your new Job:

  4. When you drop it there a list of components is displayed. Select tFileInputDelimitedand click OK:

  5. To become more familiar with the Palette please search for the tFileOutputXML component. Enter tfileoutput as a search criteria and click the magnifying glass. You will find the component under Filefolder, Output subfolder. Select it:

  6. Place it to the right of your input component:

  7. Connect the input to the output component. Right-click on StatesCodes, select Row,click Main and then link it to the tFileOutputXML:

  8. Double-click the tFileOutputXML component to open the Component tab and configure it. Click the ellipsis ... next to the field File Name:

    Set the path to C:/StudentFileswith the File name out.xml. Click Open:

  9. Select the Run Tab and click Run to execute the job.

  10. Open the File Browser and go to the folder C:/StudentFiles. Open the file out.xml with Notepad ++:
