Database Connection

Overview

Now you will use another Metadata Wizard to create a database connection. You will use it in a new Job which will be similiar to the last one. The output shall be a new table in a MySQL database. This Metadata will also be used later in the database lab to connect to this database again.

Create Database Connection

  1. Close your existing Jobs and expand again the Metadatafolder in the Repositoryand right-click on Db Connectionthen click Create Connection:

  2. Enter LocalMySQLfor the Name, enter your text for the Purposeand a Description. Click Next:

  3. In the following windows select MySQL for the DB Type and enter the data to connect to the database. Enter the following login credentials and click the button Check at the end to check the connection:
    • Login " root"
    • Password " root
    • Server " localhost"
    • Port leave it to " 3306"
    • Database " training"

  4. The Check Connection windows appears and shows a successful connection. Click OK and then click Finish:

  5. Expand the Metadataagain and expand Db Connections:

Export Items

The connection LocalMySQLwill be reused in the database lab. To secure that you will have it available and to understand how you can archive or export Jobs or other content you will export this connection as a zip file with the option to upload it.

This is a general way of exporting Jobs or Projects.

  1. You can right-click on the item you want to export or somewhere else in the Repositoryand select Export items:

  2. You mark Select archive fileand change the path to C:/StudentFiles/Db Connections.zip. The option Export Dependenciesoffers the possibility, in case you export a Job, to automatically select all dependencies like used connections etc. in order not to forget an important item. In this case you only need to export the selected item. Don't select additional items and click Finish:

    Next step will be to create a database output in a new Job.
