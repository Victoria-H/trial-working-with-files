Creating the Product Table

Overview

The next step is to create and populate the product table.

Create Job

  1. Duplicate the Job Custprep2Table, naming it Productprep2Table. When you are creating a Job that is very similar to an existing Job, it can save time to duplicate it rather than to start from scratch.

  2. Open the new Job and then close all other Jobs to avoid confusion.

Create File Metadata

  1. Create a new File delimitedmetadata, naming it RetailProducts:

  2. Enter RetailProducts for the Name and fill in text for Purpose and Description. Then click Next:

  3. Configure the metadata with the following information:

    The file is C:/StudentFiles/refProducts.csvand the Formatis Windows:

  4. There are no header rows to skip and no row of column names (no configuration needed for Step 3). Click Export as context:

  5. Select Create a new repository context and click Next:

  6. Leave the Nameas RetailProducts and enter the Purpose and Description and then click Next:

  7. The context variables and values are displayed. Then click Finish:

  8. You can see the new Metadata appeared in the Repository under File delimited section.

  9. Enter id for the first Column and set the Type to Integer and enter Product for the second Column and click Finish:


Create Schema Metadata

  1. Double-click the RetailCustomerscomponent to open the Componentview and click the button marked with an ellipsis (...)next to the Property Typetext box:

  2. Select RetailProductsfrom the Metadataand click OK:

  3. Click Edit Schemaso that you can save the schema separately:

  4. Leave the selection to View schemaand click OK:

  5. Click the Save button to Save as generic schema :

  6. Click OKto leave it in the default folder:

  7. Enter ProductSchema for the Name, and enter values into the Purposeand Descriptionboxes then click Finish followed by OK:

    Note: The schema now appears in the Repository.

    Expand the Generic schema under Metadata in the Repository until you see the word metadata. Double click on metadata, change Name to RetailProducts, and click Finish:

  8. Back to the Component view of the RetailCustomers component: in the Viewtab, change the name of the component to RetailProducts:

    Now this component reads the product data file using the appropriate schema from the Repository

Create Table

  1. Double-click the LocalMySQLcomponent to open the Componentview:

    Note:The schema is now Built-in, because you propagated the schema changes from the RetailProductscomponent.

  2. Change the schema to Repository, specify the new generic schema you just created, and then change the table name to "RetailProducts":

    Now this component will write to a table named RetailProductswith the schema taken from the Repository.

  3. Double-click the tCreateTablecomponent and change the Schemato the new ProductSchemametadata, and then change the table name to "RetailProducts":

  4. Runthe Job. The product file contains 50 rows:

Verify Results

  1. In MySQL Workbench, replace the existing text in theQuerywindow with the following command:

    select * from retailproducts;

    and then click the lightning bolt and verify the result:

    Note: Again, if the query does not return results, double-check your work and make any necessary corrections.

Next

You now have database tables containing both product and customer data. Now you can set up the shop tables.
