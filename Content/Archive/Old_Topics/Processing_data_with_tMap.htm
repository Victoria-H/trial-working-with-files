Processing data with tMap

Overview

In the Job AscOrderyou used a component out of the processing components to write your output in an ascending order.

The most powerful component in this group is tMap. To get a first impression of the options and possibilities, duplicate the Job States2MySQLand insert the tMapcomponent . Then you will process the data with an expression.

Duplicate Job

  1. Right-click on the Job States2MySQland click Duplicate:

  2. Enter tMapMySQLfor the Input new nameand click OK:

  3. Double-click the new Job out of the Repository to open it in the Workspace:

  4. You want to place the tMapcomponent between the input and the output which will then present a typical ETL Job. Select the link row1 (Main)between the two components and type tMap:

  5. You will be asked for the new Output name. Enter MySQL and click OK:

  6. The system asks Do you want to get the schema of the target component?and you click Yes:

  7. Your Job should look like this now. You may rearrange the components on the Workspace to get a clear Job:

  8. Double-click the component tMapand have a first look at the tMapdialog:

    Note: You see the fields of the input and output are automatically connected. You have several options to process your data in tMap.

  9. To output the field "StateName" with uppercase you will need to enter an expression for this field in the output table. Click the field row1.StateNameand then click the ellispsis next to it " (...)" to open the Expression Builder.

  10. The field "row1.StateName" is already entered into the expression. At the bottom you can select from different Categories and Functions. You also can add or enter the expressions manually. Position the cursor at the end of row1.StateName and type .to. You will see a list being displayed right after you started typing:

  11. Select toUppercase():String - String and it will be entered. Click Ok to close the window:

  12. The expression appears in the output table. Click OK

  13. Double-click the tMySQLOutput component and update the Action on table to Drop table if exists and create.
  14. Run the Job.

 

Run Job

Now you run the Job and examine the result in the database table.

  1. Run the Job and open the MySQL database:

  2. Expand it to the table " Statemysql":

  3. Enter the query "select * from training.statemysql" to display the results:
