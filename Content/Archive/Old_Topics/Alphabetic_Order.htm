Alphabetic Order

Overview

In this variation of the last Job you will start to use processing components. In case you want an alphabetic order for the output and no more specific transformations you can use the tSortRowcomponent. You will learn how to modify an existing Job starting again with duplicating it and redefine it afterwards. You also will pay attention to the error logs to understand how to correct your settings.

Duplicate and modify a Job

  1. Right-click on the UseMetadataJob and select Duplicate:

  2. Enter AscOrderfor the new Job (like ascending order) and click OK:

  3. The Job is displayed and you want to insert the processing component tSortRow. One way to modify your Job, is to delete the link , to place the new component between input and output and link them again. So right-click the link and select Delete:

  4. Move the component tFileOutputXMLto the right to have more space for the new component. You are looking for a "sorting" component. To find such components in the Palette you ente sort and click the magnifying glass agin and select tSortRowout of the results:

  5. Place the component tSortRowin the Workspace:

  6. You link the three components starting with a right-click on StatesCodesand select Rowthen click Mainand connect to the component tSortRow:

  7. Go on with the next link and see the result:

Place tSortRow directly

Another way to enter the new component between the input and output component is to place it directly between the two components.

To prepare this option, delete the tSortRowcomponent and connect the input and output again.

  1. Right-click on tSortRowand selectDelete:

  2. Right-click on StatesCodesselect Rowand click Mainand connect to tFileOutputXML:

  3. Select the link between the components and type tSortRowto place the new component. See also the description of the component which is displayed:

  4. The connections between the three components are set automatically:

  5. Save your Job.

Run the Job

  1. You do not see any warnings and decide to Run the Job. Select the Run tab and click Run:

  2. Open the out.xml file from the C:/StudentFilesfolder and see that the output is not ordered:

  3. To define an ascending order, double-click the component tSortRow and click the green "+" at the bottom of the Criteria table:

  4. The field StateCodeis automatically entered by the tool. Order is set to " asc" so the default order is ok. Lets assume you find this setting fine and you want to Run the Job again:

    .

  5. Select the Run tab and Run the Job again. Then verify the error message. It indicates that column StateCodeis set to " num" which is not correct. It has to be set to alpha to get an alphabetic ordering:

  6. Double-click the tSortRowcomponent again and select alphafor the column sort num or alpha?.Then Run the Job again:

  7. The Job runs without errors and displaying the out.xml file from the C:/StudentFilesfolder , shows the correct output:
