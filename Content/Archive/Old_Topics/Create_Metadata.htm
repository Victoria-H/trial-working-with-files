Create Metadata

Overview

In this lab you will learn how to duplicate a Job, how to create and use the Metadata in the actual Job and how to reuse it in other Jobs.

Duplicating a Job shows how easy it is to reuse an existing Job and create a new one based on it.

You will create the Metadata first out of the duplicated Job. In the second part you will delete it and create the Metadata directly in the Repository using the Wizard without having a Job or component prepared.

Duplicate Job

  1. Right-click on FirstJob 0.1 and click Duplicate:

  2. Enter Create_Metadatafor the Input new nameand click OK:

  3. Double-click the new Job Create_Metadataand close the FirstJob:

  4. Double-click the component tFileInputDelimitednamed Statesto open the Componenttab and click the disc in order to save this schema in the Metadatato be reused further on:

  5. The Wizard for creating this type of Metadata will pop-up. Enter States for the Nameand enter the following text for the Purposeand Description.Then click Next:

  6. The Wizard displays the next step and entries. Fileallows you to select another file for creating this Metadata. You will leave it to the States.csv file. The File Viewershows a preview of the data. Change the Format to WINDOWS and then click Next:

  7. Step 3 of the Wizard offers several options like Field Separator, Header, Footer,several CSVoptions and a Preview. Don't change the settings and click Next:

  8. Enter StateCodes for the Nameand enter text into the field Comment. Change the names for the columns in the table to StateCode and StateName , set the Lengthto 2 and 20 respectively and then click Finish:

  9. In the Repository expand the folder Metadata and display the new StateCodesmetadata:

    Note: It can now be reused in all your Jobs.

Delete and Create Metadata in the Repository

Overview

You created the Metadata for the component tFileInputDelimited. You used the wizard for creating the Metadata. There are several other wizards available for standard formats.

In case you cannot use the wizards you will create a Generic Schema by entering all the data manually.

Delete Metadata

  1. Expand the Metadata in the repository and right-click on States0.1then click Delete:

  2. The Metadata will be created again using the same name. It is still in the Recycle bin and needs to be removed, else the system will retrieve the name. Right-click on Recycle binand click Empty recycle binand answer with Yes:

  3. Right-click File delimitedand click Create file delimited:

  4. The Wizard for creating this type of Metadata will pop-up and you enter " StatesCodes" for the Nameand enter your text for the Purposeand Description:

  5. The Wizard displays the next step and entries. Serverindicates that you only have a local connection for this installation and do not use remote data. Fileallows you to select another file for creating this Metadata. You will leave it set to the " States.csv" file. The File Viewershows a preview of the data. Click Next:

  6. In step 3 of the Wizard set the Field Separatorto Commaand leave Header, Footerempty. The CSVoptions will not be used and the Previewshows the correct data. Click Next:

  7. Enter " StatesCodes" for the Nameand enter text into the field Comment. Change the names for the columns in the table to " StateCode" and " StateName" and set the Lengthto " 2" and " 20" then click Finish:

  8. In the Repository expand the folder Metadataand display the new StatesCodesMetadata:

Next

Now you are ready to create your Job again, using the Metadata directly in the Workspace.
