Creating the Join

Overview

Your Job generates data representing purchases, now you need to join that information with customer data to create a comprehensive sales report.

Read Customer Data

 

Show Me!

  1. Place a tFileInputDelimited component on the design workspace to the right of the tRowGenerator component.

  2. In the Component view, change Schema from Built-In to Repository and then use the button marked with an ellipsis (...) to set the Schema to metadata under FullCustomer (this is the schema you saved earlier from the tMap component).

  3. Set the File name to "C:/Student Files/CappedOut.csv". This is the file that the tFileOutputDelimited component elsewhere in the Job writes that contains the customer information.

  4. Change the Field separator to a comma to match the file contents and then set the Header value to "1" to account for the first row containing the column names:

  5. Change the label of the component (in the View tab) to "Full Customer Data".

Add a tMap Component

 

Show Me!

  1. Place a tMap component on the design workspace, connect the tRowGenerator component to it with a Main row, and then connect the Full Customer Data component to it with a Main row. This Main row will be automatically renamed to Lookup:

    Notice that there are two distinct areas on the design workspace: the components from the duplicated Job above, and the new components you added below. Nothing yet connects those two areas.

  2. Add a tLogRow component to log the output from the tMap component to the console, naming the row CustomerPurchases:

Configure the tMap

 

Show Me!

  1. Configure the tMap component to perform a left outer join (the default join) between row3 and row4 on the Customer ID columns (use the instructions in the previous lesson to refresh your memory, if necessary).

  2. Configure the output table (CustomerPurchases) to contain all the columns from both input tables, except for the ID column from row4(click image to enlarge):

  3. Click Okand then propagate the changes.

Connect the Subjobs

 

Show Me!

Right-click CustomersFile, click Trigger followed by OnSubjobOk, and then click tRowGenerator:

Your Job is now made up of two subjobs: the first creates a file containing complete customer data, with both the state code and the state name, and the second subjob joins the data in that file to generated purchase data to create complete purchase records. The trigger OnSubjobOk specifies that the connected subjob only run if the first subjob succeeds, making sure that the customer data file is in place before the second subjob attempts to access it.

  2. Click Clear to remove the statistic information from the design workspace.
