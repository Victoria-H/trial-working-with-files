Using Repository Context Variables

Overview

In the previous exercise, you created a built-in context variable that was only available to a single Job. Now you will create a Repository context variable that can be reused in any Job in the project.

Create Repository Context Variable

  1. CREATE A CONTEXT GROUP IN THE REPOSITORY TO ORGANISE YOUR CONTEXT VARIABLES

    In the Repository, right-click Contexts, and click Create context group:

    Note: A context group is just an organizing tool for repository variables, so it is a good idea to name it something meaningful.

  2. GIVE THE GROUP A NAME, PURPOSE AND DESCRIPTION

    Enter LocationContextVariables for the Name and then enter appropriate information into the Purpose and Description boxes.
    Then click Next:

  3. ADD A NEW CONTEXT VARIABLE

    Click the green [+] (Add) button at the bottom left of the table to add a new variable. Name it RepoOutputDir to differentiate it by name from the built-in context variable OutputDir.
    Set the default value to "C:/StudentFiles/" :

  4. ADD A NEW CONTEXT

    Click the green [+] at the top right of the table, which is the Configure Contexts button and then click New :

  5. NAME THE CONTEXT

    Enter Production and then click OK:

  6. CONFIGURE  THE NEW CONTEXT

    Click OK again.
    Then set the value for RepoOutputDir in the Production context to "C:/StudentFiles/Production/" and click Finish:

    Reminder: The Value for each context variable includes a final "/" and is enclosed in double quotes.

  7. CHECK THE GROUP EXISTS IN THE REPOSITORY

    The new group appears in the Repository, containing a single variable:

Add Repository Context Variables to a Job

  1. OPEN THE CONTEXTS VIEW FOR THE JOB Context_Uppercase

    In the Context_Uppercase Job, click the Contexts view.

  2. OPEN THE SELECT CONTEXT VARIABLE WIZARD AND SELECT THE CONTEXT

    Click the contexts button below the Variables table. The Select Context Variables wizard opens in a new window.

    Select the check box for the Context variable previously defined to specify all the variables in that group, and then click OK:

    This is where you select the variables from a group that you want to make available to the Job.

  3. YOU CAN SEE THE REPOSITORY CONTEXT VARIABLE IN THE CONTEXTS VIEW

    The variable and the group appear in the Contexts view.

    The values are the ones you set in the Repository and you cannot change them from here. You can of course edit them from the Repository directly however.

Use Repository Context Variables

  1. CHANGE THE OUTPUT PATH OF THE tFileOutputDelimited COMPONENT TO USE THE NEW CONTEXT VARIABLE

    Double-click the tFileOutputDelimited component to open the Component view.
    Update the File Name value to context.RepoOutputDir + "CappedOut.csv":

  2. SELECT THE PRODUCTION CONTEXT AS THE DEFAULT FOR THIS JOB

    Click the Contexts tab again and select Production for the Default context environment:

     

  3. RUN THE JOB

    Change to the Run view and run the Job.

    The file created during this run of the Job appears in the C:/StudentFiles/Production directory because of the default context variable, which was set to Production.

  4. OPEN THE REPOSITORY CONTEXT FOR EDIT

    In the Repository, under Contexts, double-click LocationContextVariables to open the group for edit.
    Then click Next :

  5. MODIFY THE VARIABLE VALUE

    Change the value under Production to C:/StudentFiles/New_ Production/ and click Finish:

  6. PROPAGATE CHANGES

    When asked if you want to propagate changes, click Yes:

    Note: this prompt is asking you if you want the change to be reflected in all Jobs that use the Repository variable.

  7. CHOOSE THE JOBS TO APPLY THE MODIFICATION TO

    The Update Detection window pops-up. This lists the items in a particular Job that use this context variable. If you had multiple Jobs using the variable, you would see all of them listed so that you could choose which one to change.

    Leave all items selected and click OK:

  8. RUN THE JOB

    Run the Job using the Production context again. Note that CappedOut.csv file is now created in the new directory, as you would expect from changing the value of the context variable.

    With the context variable set in the Repository, you can now use the variable in any Job in the project.
