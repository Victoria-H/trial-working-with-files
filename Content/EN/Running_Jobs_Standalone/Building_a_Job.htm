Building a Job

Overview

In an earlier lesson, you exported both parent and child Jobs. These Jobs can be shared with others, but still only run within the context of the Talend Studio. In many situations, you will need to run a Talend Job independent of the Talend Studio. In order to do that, you need to build and export the Job.

Build Job

  1. OPEN THE Build Job WIZARD

    Right-click the FilterStates_tMap Job in the Repository and click Build Job:

  2. NOTE THE OPTIONS IN THE WIZARD

    This option allows you to export the Job in a variety of formats with several options.
    The following wizard appears:

  3. CHOOSE THE  PATH FOR THE ARCHIVE FILE YOU WILL CREATE

    Click the Browse button, navigate to C:/StudentFiles and leave Filter_States_tMap.zip for the File name. Then click Save:

  4. SELECT/DESELECT THE OPTIONS YOU NEED

    Leave the Build type set to Standalone Job and clear the Context scripts and Java sources check boxes.
    Then click Finish:


    The Context scripts option is useful if you created context variables in your Job and you want to have them written in a configuration file.
    The Java sources option is useful if you need the Java source files in order to recompile them outside of the Talend Studio, in another development environment.

Extract and Run

  1. EXTRACT THE BUILD FOLDER YOU JUST CREATED

    Using Windows Explorer, locate the file you just created, FilterStates_tMap.zip, and then extract the content.

  2. LOCATE THE WINDOWS BATCH FILE

    Locate the file FilterStates_tMap_run.bat (typically placed in a Filter_States_tMap sub-folder).


    Note that a Linux shell script file is in the same directory.

  3. EXECUTE THE FILE TO RUN THE JOB

    Double-click FilterStates_tMap_run.bat to execute it.

    A command window opens briefly as the Job executes, but because the Job has no visible results, it is not easy to see what happened.
    However, recall that this job was generating three output files : CA_Out.csv, AK_Out.csv and NY_Out.csv. Check these files have been created in the C:/StudentFiles folder, and have a date and time associated with them which correspond to the FilterStates_tMap_run.bat execution time.
