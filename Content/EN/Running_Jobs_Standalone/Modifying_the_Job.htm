Modifying the Job

Overview

When you are working on a large project, you may want to save the current version of a Job before making changes in case you need to revert.

Create a New Version

  1. OPEN ANOTHER VERSION OF THE FilterStates_tMap

    Close the FilterStates_tMap Job. Right-click the Job in the Repository and then click Open another version:

  2. NOTE THE EXISTING VERSIONS OF THE JOB

    The Open another version wizard is displayed:

    From this window you can open an existing version or create a new one.

  3. CREATE AND OPEN A NEW VERSION OF THE JOB

    Select the Create new version and open check box and click the button marked with a capital M. Note the version number changes from 0.1 to 1.0. Then click Finish:

    The button marked with a capital M increases the major number, the button marked with a lower-case m increases the minor number.

  4. NOTE THE NEW VERSION NUMBER

    In the Job view you can now see the new version number:

  5. VERSION INFORMATION IS AVAILABLE IN THE Version SECTION OF THE Job VIEW:

    Also note the version numbers in the Version section of the Job view:

Modify the new version of the Job

  1. CHANGE THE STATE FILTER

    Double-click the tMap component and change the filter criteria for the CappedOut table to "AL".equals(row2.StateCode) and click OK:

  2. CHANGE THE OUTPUT FILE

    Double-click the output component CA to open theComponent view and change the file name to "C:/StudentFiles/AL_Out.csv"

  3. CHANGE THE OUTPUT COMPONENT NAME ACCORDINGLY

    Click the View section and change the Name to AL:

  4. SAVE THE MODIFIED JOB

    Save the Job but do not run it.

Build and Run

  1. BUILD THE NEW VERSION OF THE JOB

    Build the Job again. Make sure you use version 1.0:

  2. RUN THE BUILT JOB AND CHECK THE RESULTS

    Extract the contents of the archive, run the script, and then examine the new output file. (If you extract to the same folder you will need to overwrite the old contents.) The fact that the new file was created is tangible evidence that the Job executed successfully.
