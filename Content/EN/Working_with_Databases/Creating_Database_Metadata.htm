Creating Database Metadata

Overview

Now you are ready to create metadata regarding the connection to your local database server and the specific database. Creating metadata with this information makes it easier to reference it in many components in several Jobs.

Create Database Connection Metadata

  1. CREATE DATABASE METADATA IN THE REPOSITORY

    In the Repository, under Metadata, right-click Db Connections and then click Create connection :

    Note: Because many components in your project need to connect to the same database, storing the connection information in the Repository saves you from having to enter the information repeatedly for each component.

  2. GIVE THE METADATA A NAME, PURPOSE AND DESCRIPTION

    Enter LocalMySQL into the Name box, provide a Purpose and a Description. Then click Next:

Configure Database Metadata

  1. ENTER THE DATABASE CONNECTION DETAILS

    Select MySQL out of the drop-down list for the in the DB Type. Then configure the connection as follows :
    Login: root
    Password: root
    Server: localhost
    DataBase: training

    Then click Check:

    Note: this configuration information is specific to the training environment, where a MySQL database is set up for you. In your own environment, you would, of course, provide the connection information appropriate for your configuration.

  2. CHECK CONNECTION IS SUCCESSFUL

    The Check should be successful and click OK:

    If the connection is not successful, correct the connection information and then try again.

    3. Note: The first time you configure a connection in your own environment, you may be prompted to download and install an external jar file. If this happens to you, install the file and continue. All such required jar files are already installed in the training environment.

Export as context

As you did for the RetailCustomer metadata, you will create context variables with all the information about your connection to MySQL database.

  1. EXPORT THE DATABASE CONNECTION INFORMATION AS A CONTEXT

    Click Export as Context and then click Next:

  2. CREATE A NEW CONTEXT GROUP

    In the Create / Reuse a context group wizard, click Next:

  3. CHECK THE CONTEXT VARIABLES THAT ARE TO BE CREATED AND FINISH

    In the Variables tab, you will see the list of context variables created and in the Value as Table tab, you will see the default value of each context variable.
    Then click Finish:

  4. FINALISE THE DATABASE CONNECTION METADATA

    Going back to the Database connection window, notice that instead of the values you typed in each box, you will have context variables.
    Then click Finish:
