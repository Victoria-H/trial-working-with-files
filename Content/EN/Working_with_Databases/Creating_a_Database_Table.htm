Creating the Customer Table

Overview

Your next step is to connect to an existing database, create a table for customer data, and then load data from a text file into that database table.

Create new Job

  1. CREATE A STANDARD JOB IN THE REPOSITORY

    Right-click the Sales folder under Job Designs / Standard and select Create Standard Job:

  2. GIVE THE JOB A NAME, PURPOSE AND DESCRIPTION

    Enter CustPrep2Table for the Name, fill in the Purpose and Description and click Finish:

Create a Table

You are going to create a database table using the connection and customer file information you stored as metadata in the repository.

  1. CHOOSE A COMPONENT TO CREATE A DATABASE TABLE

    In the CustPrep2Table Job, place a tCreateTable component :

    As its name implies, this component creates a table in a database.
    A window may appear providing the option to install additional modules. These might be useful under other circumstances and in your home environment you should examine the list carefully and read about the modules to decide if you want to install any of them. Note that none are marked as Required.

    If this window appears, select the checkbox in front of Do not show again and then close the window.

  2. OPEN THE COMPONENT VIEW FOR tCreateTable AND THEN THE REPOSITORY CONTENT WIZARD

    Double-click the tCreateTable component to open the Component view.

    Select Repository in the Property Type list to use the stored database connection information.
    Then click the button marked with an ellipsis [...] next to the field Property Type

  3. SELECT THE DATABASE CONNECTION YOU CONFIGURED BEFORE

    In the Repository Content wizard, select LocalMySQL and then click OK:

  4. ADD THE CONTEXT PARAMETERS TO THE JOB

    Then click Yes in order to add context parameters to the Job:

  5. CHECK THE CONFIGURATION

    Check the configuration of the tCreateTable component:

    The information from the stored connection now appears in the Component view.

  6. TELL THE COMPONENT TO USE A SCHEMA FROM THE REPOSITORY

    Set the Schema to Repository (changing it from Built-In) and then click the button marked with an ellipsis (to the left of Edit schema) to choose the schema:

  7. SELECT THE CustomerMetadata SCHEMA YOU CREATED EARLIER

    Under Metadata click File Delimited and under RetailCustomers, select CustomerMetadata. Then click OK:

    Recall that this is the Customer Schema you defined in the Repository earlier.

  8. SET THE TABLE NAME AND TABLE ACTION

    Select Drop table if exists and create out of the list for Table Action.

    Set the Table Name to "RetailCustomers":

    Now this component is configured to connect to your local MySQL server and create a table named RetailCustomers with the stored RetailCustomers schema in the training database ( this database was created for you in this environment). Hint: Initially, the Table Name is blank. When you set the Schema to the Repository (RetailCustomers) it gets set to "CustomerMetadata". Be sure that you changed it to "RetailCustomers" last (before continuing with the rest of the steps below).

    9. The Table Action parameter set to Create table by default, can only happen once. By changing the action to Drop table if exists and create, you can run this Job repeatedly if necessary without causing an error.

     

Populate the Customer Table

You are going to put the data from the C:/StudentFiles/refCustomer.csv file into your database table. You have already created metadata for this file and for the database connection.

  • You have a component, tCreateTable , that creates your database table.
  • When the table is created, it will trigger an input component to read the information from your csv file.
  • Then you will choose a component that writes this input to your database table.

  1. STARTING FROM THE FILE METADATA IN THE REPOSITORY, ADD THE tFileInputDelimited COMPONENT TO THE WORKSPACE

    Drag the RetailCustomers metadata into the design workspace, choosing tFileInputDelimited as the component. Place the component under the tCreateTable component.

    This file input component reads the customer data file. As you exported your information as context variables, you will be prompted to add context parameters to the Job.
    Click Yes:

  2. CHECK THE COMPONENT IN THE COMPONENT VIEW

    Double-click RetailCustomers to open the Component view. The information displayed is just what you entered in the Repository metadata and saved as context variables:

  3. TRIGGER THE INPUT COMPONENT FROM THE tCreateTable COMPONENT

    Connect the tCreateTable component to the RetailCustomers component with an OnComponentOK trigger:

    This trigger specifies that the connected subjob only executes when the component runs without error. In other words, the Job will not populate the table unless the table is successfully created first.

  4. STARTING FROM THE DATABASE CONNECTION METADATA IN THE REPOSITORY, ADD A COMPONENT TO WRITE THE DATA TO YOUR DATABASE

    Drag the database connection LocalMySQL from the Metadata onto the design workspace to the right of RetailCustomers:

  5. CHOOSE THE tMysqlOutput COMPONENT

    Click tMysqlOutput in the Components window (to write to the database) and then click OK:

  6. OPEN THE COMPONENT VIEW OF LocalMySQL

    Double-click the new component LocalMySQL to open the Componentview.
    Notice that the connection information is already set based on the metadata.

  7. SPECIFY THE SCHEMA

    Specify the Repository schema RetailCustomers - CustomerMetadata (as you did with the tCreateTable component). Set the table name to "RetailCustomers".

    This is the name of the table the tCreateTable component creates, so this component writes data to that new table.

  8. LINK THE INPUT COMPONENT TO THE DATABASE CONNECTION

    Right-click RetailCustomers and click Row , then Main and then drag it to the LocalMySQLcomponent to connect them:

  9. PROPAGATE CHANGES

    Click Yes if this message appears:

  10. RUN THE JOB

    Run the Job and then examine the results:

    The customer data file contains 5000 rows.

  11. CHECK THE WARNINGS

    Note that a warning appear on tCreateTable and LocalMySQL. Hover the exclamation mark to read the warnings :

  12. CHANGE THE SCHEMA TO BUILT-IN

    In order to solve the warning, double-click on the tCreateTable component.
    Click on the Edit Schema [...] button, select Change to built-in property and then click OK:

  13. RESET DATABASE TYPES

    The schema of the tCreateTable is then displayed.
    Note that the DB Type column is empty, and highlighted in orange.
    Click the Reset DB Types button and then click OK:

  14. CHECK

    The DB Type column is then filled in:

  15. NOTE THE WARNINGS HAVE DISAPPEARED

    Click OK and note that the exclamation mark disappeared for tCreateTable:

  16. REMOVE THE WARNINGS FROM tCreateTable IN THE SAME MANNER

    The LocalMySQL warning is also related to the fact that the DB Type is not defined in the schema. In order to remove the warning on this component, repeat the steps you executed to remove the warning on the tCreateTable component.

Verify Results

You are going to check that the database table has been created.

  1. OPEN THE MySQLWORKBENCH

    From the Windows Start menu, click All Programs, followed by MySQL, and then MySQL Workbench 6.3 CE to start the program mySQL Workbench.
    The welcome window appears briefly, and then the Workbench Central Workspace appears:

  2. OPEN THE CONNECTION

    Double-click Local instances MySQL56 to open the connection to start querying.

  3. ENTER THE USERNAME

    Enter root into the Username box and then click OK:

  4. ENTER THE PASSWORD

    Enter root into the Password box and then click OK.

    The SQL Editor opens.

  5. SPECIFY THE DATABASE

    Double-click training in the Object Browser on the left to specify the database you are connecting to and using for your queries.

  6. QUERY THE DATABASE

    In the Query window enter the following command, including the semi-colon:

    select * from retailcustomers;

  7. EXECUTE THE QUERY

    Click the lightening bolt to execute the query:

    This shows the rows from the table, demonstrating that your Job loaded data into the table. If your query does not return any results, double-check your work in Talend Studio and make any necessary corrections. Make sure that the table name matches in all components.

  8. Leave MySQL Workbench running to use in the next exercise.
