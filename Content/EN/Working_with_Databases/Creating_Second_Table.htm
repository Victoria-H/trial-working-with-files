Creating the Product Table

Overview

The next step is to create and populate the product table.

This procedure is very similar to to the Custprep2Table Job so you can duplicate it and make the necessary changes.

Duplicate Job

  1. DUPLICATE THE JOB Custprep2Table

    Duplicate the Job Custprep2Table, naming it ProductPrep2Table.
    When you are creating a Job that is very similar to an existing Job, it can save time to duplicate it rather than to start from scratch.

  2. OPEN THE JOB

    Open the new Job and then close all other Jobs to avoid confusion.

Create and Populate the Product Table

  1. CHANGE THE tCreateTable SCHEMA

    Double-click the tCreateTable component.
    Change the Schema to RetailProducts - ProductMetadata.
    Then change the Table Name to "RetailProducts":

  2. SELECT THE CORRECT METADATA FOR THE tFileInputDelimited COMPONENT

    Double-click the RetailCustomers component to open the Component view.
    Click the [...] button after Property Type and select RetailProducts from the File delimited Metadata:

    Answer Yes in the following pop up window:

  3. CHANGE THE SCHEMA

    Set the schema to RetailProducts - ProductMetadata

    Answer Yes when asked if you want to propagate the changes.

  4. CHANGE THE COMPONENT NAME

    Go to the View section and change the name of the component to RetailProducts:

  5. OPEN THE COMPONENT VIEW OF THE OUTPUT COMPONENT

    Double-click the LocalMySQL component to open the Component view:

    The schema is now Built-in, because you propagated the schema changes from the RetailProducts component.

  6. CHANGE THE SCHEMA

    Change the schema to Repository, specify the RetailProducts - ProductMetadata schema, and then change the table name to "RetailProducts":

    Now this component will write to a table named RetailProducts with the schema taken from the Repository.

  7. RUN THE JOB

    Run the Job. The product file contains 50 rows:

  8. CHECK THE WARNINGS

    Note that a warning appear on tCreateTable and LocalMySQL. Hover the exclamation mark to read the warnings :

  9. RESOLVE THE WARNING ON THE tCreateTable COMPONENT IN THE SAME WAY AS FOR THE Custprep2Table JOB

    In order to solve the warning, double-click on the tCreateTable component.
    Click on the Edit Schema [...] button, select Change to built-in property and then click OK:

  10. RESET THE DB Type COLUMN

    The schema of the tCreateTable is then displayed.
    Note that the DB Type column is empty.
    Click the Reset DB Types button and then click OK:

  11. CHECK

    The DB Type column is then filled in:

  12. NOTE THE EXCLAMATION MARK HAS DIDAPPEARED

    Click OK and note that the exclamation mark disappeared:

  13. RESOLVE THE WARNING ON THE OUTPUT COMPONENT

    The LocalMySQL warning is also related to the fact that the DB Type is not defined in the schema. In order to remove the warning on this component, repeat the steps you have just executed to remove the warning on the tCreateTable component.

Verify Results

  1. USE THE MySQL WORKBENCH TO CHECK THE TABLE IS POPULATED

    In MySQL Workbench, replace the existing text in the Query window with the following command:

    select * from retailproducts;

    and then click the lightning bolt to verify the result:

    If the query does not return results, double-check your work and make any necessary corrections.

Next

You now have database tables containing both product and customer data. Now you can set up the shop tables.
