Detailed Output

  1. ADD ANOTHER MAPPING COMPONENT AND CONNECT IT

    Add another component tXMLMap to the existing Job and right-click on tLogRow_1 to connect it to the newly added component with a Main row:

  2. ADD ANOTHER tLogRow COMPONENT AND CONNECT IT

    Enter another tLogRow component and connect it to the second tXMLMap with a *New Output*(Main) row:

  3. NAME THE OUTPUT

    Name the new output Details and click OK:

    Similar to before, the second tXMLMap is indicating an error because the schema column data is not yet defined.

  4. CONFIGURE THE SECOND tXMLMap COMPONENT

    Double-click the second tXMLMap component, from the input table, right-click on payload and select Import From Repository :

  5. LOCATE THE GetCityWeatherByZIPResponse0.1

    Open the Metadata folder and the following sub-folders until you locate the entry GetCityWeatherByZIPResponse0.1.
    Select it and then click OK:

  6. SELECT THE RESPONSE

    Have a look at the response message structure. Select the entire response.

    To do so click on tns:Success, press the SHIFT button on the keyboard, hold it and then click on tns:Remarks to select all response elements:

  7. PUT THE RESPONSE ELEMENTS IN THE OUTPUT TABLE

    Drag and drop these elements to the Details output table. You will need to change the name for every column one by one by removing the leading ":".
    Then click OK for every element that is displayed. For example, the Success column:

    Again, you will have to repeat the action for all 14 columns.

  8. CHECK THE MAPPING

    The mapped elements after editing all Column names should look as follows :

    Click OK after verifying the column names.

  9. SYCHRONISE THE tLogRow_2 SCHEMA

    Double-click the tLogRow_2 component and then click the button Sync columns.

    In order to verify the sync click the [...] button in front of Sync columns:

     

  10. CHECK THE tLogRow_2 SCHEMA

    The schema will be displayed. Each of the Input table columns should be mapped to the Output table columns.

    Check that the columns have been synchronized and then click OK:

Deactivate element

It is important to know that elements can be deactivated and activated as needed.

  1. NOTE WHAT HAPPENS WHEN YOU DEACTIVATE A COMPONENT

    Deactivate the tLogRow_1 component from the Job.
    Right-click on the element tLogRow_1 then Deactivate tLogRow_1:

    The component becomes transparent in the design space.

  2. CHECK THE OUTPUT

    Run the Job again and verify the output:

    Note that tLogRow_1 XML output is no longer displayed in the execution console.

  3. NOTE THE FUNCTION OF THE Wrap CHECKBOX

    Deselect the Wrap check box and notice the change in the displayed result. You have to scroll horizontally to see the response data because it is no longer wrapped:

  4. REACTIVATE THE tLogRow_1 COMPONENT

    Right-click on tLogRow_1 and click Activate tLogRow_1.

    If you run the Job again, you will see both tLogRow_1 and tLogRow_2 in the output once again.
