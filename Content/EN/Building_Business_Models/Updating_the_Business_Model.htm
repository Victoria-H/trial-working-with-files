Updating the Business Model

Overview

A business model is only useful if it is kept up-to-date, so it is a good idea to revisit it frequently to make sure all relevant pieces are represented. In this section you will also add external documentation from your local file system into the Repository and then include it as part of your graphical Business Model.

Add Project Plan

  1. ADD DOCUMENTS TO YOUR PROJECT

    In the Repository, right-click Documentation and then click Add documentation:

    This is how you add documentation files to your project.

  2. LOCATE THE FILE YOU WISH TO ADD

    Click Browse and then locate the file ProjectPlan.txt under C:/StudentFiles. Enter appropriate text into the Purpose and Description fields:

    Click Finish when ready.

  3. ADD ANOTHER DOCUMENT

    In the Repository, right-click Documentation again and then click Add documentation:

  4. AGAIN, SELECT THE FILE AND GIVE IT A NAME PURPOSE AND DESCRIPTION

    Click Browse and select the file C:/StudentFiles/ProjectPlan_Input.txt, enter DataInput for the Name and fill in the text for Purpose and Description fields:

    Click Finish when ready.

Associate Items

  1. RETURN TO YOUR BUSINESS MODEL

    Return to the CustCleanup Business Model created earlier.

  2. PLACE A SHAPE FOR YOUR PROJECT PLAN

    Place a Document shape on the left, naming it Project Plan:

  3. ASSOCIATE THE DOCUMENTATION WITH THE SHAPE

    Drag ProjectPlan.txt from the Repository onto the Project Plan shape:

    Tip: Note that hover-text for Documentation in the Repository displays the Description you added earlier.

  4. SEE YOUR ASSOCIATED DOCUMENT IN THE  Assignment TAB

    In the BusinessModels view below the design workspace, click the Assignment tab:

    5. This shows you what is associated with a particular shape, and allows you to add a comment as well.

  5. ASSOCIATE A JOB WITH A SHAPE

    In the Repository > Job Designs > Standard > Customers, drag LookupState onto the Join shape and then click the Assignment tab view:

    You can assign pretty much any item from the repository to a shape in a business model. You can assign multiple items to a single shape, and assign the same item to multiple shapes. This makes it easy to use the business model as a central location for documenting and accessing the project.

  6. CHECK WHICH BUSINESS MODEL ELEMENTS A DOCUMENT IS ASSIGNED TO

    Drag DataInput from the Repository > Documentation onto the Customer Data shape and then onto the State Codes shape. Right-click DataInput under Documentation and select Find Assignments:

    You can see that it has 2 assignments now, outlined with a thick orange border above. Note that these borders are not permanent, they are not added to the Business Model. Rather they only highlight the assignments for the selected document.

  7. FIND ASSIGNMENTS FOR ProjectPlan.txt

    In the Repository > Documentation, right-click ProjectPlan.txt under Documentation and then click Find Assignments:

    Now the Project Plan is highlighted and the Customer Data and State Codes are not.

    Tip: If the name of a document is not very descriptive, you can view them directly from the Studio. To do so, double-click ProjectPlan.txt in the Repository to open a read-only copy of the document in the workspace.

  8. SAVE

    Save the business model.
