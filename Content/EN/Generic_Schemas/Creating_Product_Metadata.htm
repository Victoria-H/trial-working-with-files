Create Product Metadata

Overview

Your next step is to create Metadata for the products file so that you can use it to configure a component.
You will also export Metadata information as context variables which is the best way to manage your variables.

You are going to follow the same steps as for the customers file. You are going to use the fileC:/StudentFiles/refProducts.csv. Look at it now.

Create File Metadata

  1. CREATE A NEW File delimited METADATA IN THE REPOSITORY

    Create a new File delimited metadata, naming it RetailProducts:

  2. GIVE THE METADATA A NAME, PURPOSE AND DESCRIPTION (Step 1 of 4)

    Enter RetailProducts for the Name and fill in text for Purpose and Description.
    Then click Next:

  3. DEFINE THE FILE PATH AND ITS FORMAT (Step 2 of 4)

    Configure the metadata with the following information:

    The file is C:/StudentFiles/refProducts.csv and the Format is Windows:

  4. ENTER THE FILE PROPRTIES AND CHECK THE PREVIEW (Step 3 of 4)

    There are no header rows to skip and no row for column names (no configuration needed for Step 3).
    Click Export as context:

  5. EXPORT THE INFORMATION IN A NEW CONTEXT GROUP

    In the Create/Reuse a context group wizard, click Next.

  6. GIVE THE CONTEXT A NAME, PURPOSE AND DESCRIPTION

    Leave the Name as RetailProducts and enter the Purpose and Description and then click Next:

  7. CHECK THE CONTEXT VARIABLES THAT ARE TO BE CREATED IN YOUR CONTEXT GROUP THEN FINISH

    The context variables and values are then displayed:

    Click Finish but don't close the main wizard yet.
    You still need to finalize the File Delimited metadata creation, by defining the Product schema in the next section.

Configure Product Schema

You are now back in the metadata creation wizard.

  1. MOVE ON TO THE NEXT STEP

    Click Next in File - Step 3 of 4

  2. GIVE THE METADATA A NAME AND CORRECT THE SCHEMA DERIVED BY THE STUDIO

    Set the Name to ProductMetadata.
    Enter id for the first Column and set the Type to Integer with Length 3.
    Enter Product for the second Column, leave its Type as String with Length 10.
    Then click Finish:

  3. CHECK THE METADATA IN THE REPOSITORY

    Verify that the metadata has been created in the Repository:
