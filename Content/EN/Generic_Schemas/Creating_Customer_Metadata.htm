Create Customer Metadata

Overview

Your next step is to create Metadata about the customer file so that you can use it to configure a component. You will also export Metadata information as context variables which is the best way to manage your variables.

You are going to use the file C:/StudentFiles/refCustomer.csv

Look at this file now.

Create File Metadata

  1. CREATE FILE DELIMITED METADATA IN THE REPOSITORY

    In the Repository, under Metadata, right-click File delimited and then click Create file delimited.
    You are creating metadata about the customer file. You already created similar file delimited metadata in an earlier training lesson:

  2. GIVE THE METADATA A NAME, PURPOSE AND DESCRIPTION (Step 1 of 4)

    Name the metadata RetailCustomers, provide a purpose and description and click Next:

  3. SELECT THE FILE PATH AND FORMAT (Step 2 of 4)

    Click Browse to locate the file C:/StudentFiles/refCustomer.csv, change the Format to WINDOWS, and then click Next:

Configure File Delimited Metadata

  1. ENTER THE FILE PROPERTIES AND USE Preview TO CHECK (Step 3 of 4)

    In the Rows To Skip section, set Header to 5, indicating that the first five rows of the file do not contain data.
    Under Preview, select Set heading row as column names to specify that the first row of data in the file are actually the column names and click Refresh Preview.

    Note: The Header value was increased to 6 after checking the box Set heading row as column names.


    Now the Preview area displays the contents of the customer data file in a easy-to-read format.

  2. EXPORT AS CONTEXT

    Before finalizing the metadata information for creation, you will export all the information as context variables.
    Click Export as context button.

  3. CREATE A NEW REPOSITORY CONTEXT

    On the Create / Reuse a context group wizard, click Next.

  4. GIVE THE CONTEXT GROUP A NAME, PURPOSE AND DESCRIPTION

    Leave RetailCustomers for the Name and enter text for the Purpose and the Description.

  5. CHECK THE CONTEXT VARIABLES THAT ARE TO BE CREATED IN YOUR CONTEXT GROUP THEN FINISH

    Click Next and all the context variables that will be created are displayed:

    Click Finish but don't close the main wizard yet. You will configure the Customer schema in the following section.

  6. THE NEW CONTEXT SHOULD NOW APPEAR IN THE REPOSITORY

    In your Repository, below Contexts, you can already see your new RetailCustomers context.
    All information about your file, such as the file name, field separator or the number of header lines, are now stored in context variables.

    Now, it's time to finalize the File Delimited metadata creation.

Configure Customer Schema

You are now back in the metadata creation wizard.

  1. MOVE ON TO THE FINAL STEP (Step 4 of 4)

    Click Next in the File - Step 3 of 4page of the wizard.

  2. GIVE THE METADATA A NAME AND CORRECT THE SCHEMA DERIVED BY THE STUDIO

    Change the name to CustomerMetadata and then make the following changes to the schema:

    • Specify the column id as the key.

    • Change the length of CustomerAddress to 35.

    • Configure all the columns with names that begin with Sum (Sum1 to Sum7) as type Float with a Length of 11.

    • Set the RegTime column as type String with a Length of 19.

    • Set the column RegisterTime as type Date with the format "yyyy-MM-dd hh:mm:ss.SSS"

    Review File - Step 4 of 4 in the wizard ( screen shot below) and then click Finish when ready:

    Talend Studio derived the schema from the data, and you have now corrected the schema to be accurate.
