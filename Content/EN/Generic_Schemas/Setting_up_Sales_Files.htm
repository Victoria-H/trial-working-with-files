Create a Generic Schema & Generate Data Files

Objective

In this lesson you will create a generic schema in the Repository. This schema will describe sales data information. The generic schema will then be used to generate sales data files.

Create the Generic Schema

  1. CREATE A GENERIC SCHEMA

    In the Repository view, under the Metadata folder, right-click on Generic schemas and then click Create generic schema:

  2. ENTER THE DETAILS

    In the first step, enter ShopMetadata for the Name.
    Then fill in the Purpose field with the following text: sales data.
    Enter Sales data for database lab in the Description field.
    Then click Next:

  3. CONFIGURE THE COLUMNS OF THE SCHEMA

    In the second step, set the Name to Shopmetadata.
    Then click the green [+] (Add) icon four times to add 4 columns to the table.
    Name the first column ShopName, select String for the Type and enter 10 for the Length.
    For the next columns, name them CustID, ProductID, and Quantity and select Integer for theType.

  4. CHECK AND FINISH

    Then click Finish. The final schema in the Repository should look as follows:

Generate Sales data files

You are going to use the generic schema and generate some random sales data for 3 shops to work with in your job

  1. CREATE A NEW FOLDER IN STANDARD JOB DESIGNS

    Right-click on Standard under Job Designs in the Repository and select Create folder.

  2. GIVE THE FOLDER A NAME

    Set the Label to Sales and then click Finish:


  3. CREATE A STANDARD JOB IN YOUR NEW FOLDER

    Right-click on the Sales folder under Job Designs / Standard and click Create Standard Job:

  4. GIVE YOUR JOB A NAME, PURPOSE AND DESCRIPTION

    Enter SalesFilesPrep for the Name .

    Set the Purpose to create sales data, a text of your choice for the Description and then click Finish:

  5. CHOOSE A COMPONENT TO GENERATE RANDOM DATA

    The newly created Job opens in the design workspace.
    Type tRowGenerator to add this component to the Job:

  6. TELL THE tRowGenerator COMPONENT TO USE A SCHEMA FROM THE REPOSITORY

    Open the Component view and select Repository for the Schema
    Then click the [...] icon next to the field Schema:

  7. ASSOCIATE YOUR GENERIC SCHEMA WITH THE tRowGenerator COMPONENT

    Open the folder Generic schemas and the folder ShopMetadata and then select the entry Shopmetadata.
    Click OK:

  8. OPEN THE tRowGenerator SCHEMA TO CONFIGURE IT

    Double-click the tRowGenerator component. The schema should look familiar to you. Enter 5000 for Number of Rows for RowGenerator to create 5000 entries.

  9. GIVE YOUR SHOP A NAME

    Select the column ShopName and then enter "Shop1" for the Value in the table below.

  10. TELL THE COMPONENT TO GENERATE RANDOM DATA FOR THE CUTOMER ID

    Select the CustID column and choose Numeric.random for Functions.

    In the Function parameters tab below, enter 1 for min value and leave 100 for max value:

    This will generate a random number from 1 to 100 for CustID.

  11. TELL THE COMPONENT TO GENERATE RANDOM DATA FOR PRODUCT ID AND QUANTITY

    Configure the ProductID to generate a random number from 1 to 80 , and then Quantity to generate a value from 1 to 1000 and check the final settings. Then click OK:

  12. ADD A COMPONENT TO WRITE YOUR GENERATED DATA TO A FILE

    Add a tFileOutputDelimited to the canvas:

  13. CONNECT THE tRowGenerator TO THE tFileOutputDelimited :

    Right-click the tRowGenerator component then click Row and then Main to connect it to the output component:

  14. TELL THE tFileOutputDelimited COMPONENT WHERE TO SAVE THE FILE

    Double-click the component tFileOutputDelimited to open the Component view.

    Then click the ellipsis next to the field File Name and set the value to "C:/StudentFiles/SalesFiles/shop1.csv".
    Leave the Field Separator to the default ";".

    Note: Be sure to put the output file under the subdirectory SalesFiles.

  15. GIVE THE tFileOutputDelimited COMPONENT A LABEL :

    In the Component tab click the View section and enter Shop1out in the Label format field:

  16. OPEN THE SUB JOB COMPONENT VIEW

    Left-click anywhere in the sub Job frame in the main design workspace.
    Then click on the Component view to open the sub job’s settings:


  17. NAME THE SUB JOB

    Select the check box Show subjob title in the Basic settings section.
    In the Title box, enter Shop1:

    The subjob’s title appears on top of the subjob in the design space.

  18. SAVE YOUR JOB

    Save your Job by using Ctrl + S.
