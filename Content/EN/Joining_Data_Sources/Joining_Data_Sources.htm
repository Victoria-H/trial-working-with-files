Creating a Join

Overview

In this section, you will first duplicate the Job created in the previous lesson.

Then you will use the file containing state codes and names as a lookup table to include the full state name in the output.

Duplicate the Job

  1. DUPLICATE THE UPPER CASE JOB

    In the Repository, under Job Designs, right-click UpperCase and then click Duplicate:

  2. ENTER A NEW NAME

    Enter LookupState for the Input new name and click OK

  3. OPEN THE JOB

    Double-click LookupState in the Repository to open it:

    Take a moment to examine the components to remind yourself of the function of the current Job.

Add Second Source

  1. USE THE ITEM YOU CREATED UNDER Metadata AS THE SECOND INPUT SOURCE

    Drag the new StateCodes item from under Metadata in the Repository onto the design workspace above the tMap component.
    Hint    : If you don't have room, you can left-click the group of components and simply drag and drop them lower in the design workspace.

  2. CHOOSE A COMPONENT TO USE WITH THE METADATA

    This window allows you to choose which component you want to use with the metadata.
    Choose tFileInputDelimited and then click OK:

  3. Your Job should look like this:

    Notice that the component label is the same as the name of the metadata item. When you store configuration information as metadata in the Repository, you can use that metadata as the starting point, as you did here, rather than begin by placing a component and then adding the configuration.

  4. JOIN THE NEW COMPONENT TO tMap

    Right-click StateCodes, click Row followed by Main, and then click the tMap component to create a row:

    Note that the row is a Lookup, not a Main. You can provide multiple input sources for a tMap component, but only one can be the Main row. The others are Lookups.

  5. OPEN THE COMPONENT VIEW

    Double-click StateCodes to open the Component view:

    Note that the Property Type and Schema value is Repository and not Built-In.

  6. COMPARE Built-In AND Repository CONFIGURATION

    Double-click the CustomersFile component to see that this component uses Built-In configuration information.
    So, Built-in property type is specific to a single component, while Repository property type is stored as Metadata and can be used by multiple components in multiple Jobs.

Configure tMap

  1. OPEN THE tMap COMPONENT MAPPING EDITOR

    Double-click the tMap component:

    Note that there is now a second input table on the left, called row2, corresponding to the input row connecting StateCodes to the tMap component.

  2. LINK THE ELEMENTS YOU WISH TO COMPARE

    Drag State from the row1 table to the Expr. key field for StateCode in the row2 table:

  3. This creates the join, so that the value of the State column from row1 will be compared to the value of the StateCode column in row2

  4. LINK StateName TO THE OUTPUT TABLE

    Drag StateName from the row2 table to the bottom of the CappedOut table on the right:

    This adds the value of the StateName column to the output.

  5. Click Ok

  6. PROPAGATE CHANGES

    The system will ask you if you want to propagate the changes. Click Yes:

  7. SAVE

    Save the Job

Run Job

  1. RUN THE JOB

    Run the Job.

  2. EXAMINE THE OUTPUT FILE

    Examine the output file.

    Right-click on tFileOutputDelimited component and select Data viewer.
    Note that not all rows include the state name:

    This means that the Job is not doing exactly what you intended, so you need to investigate further.

  3. CLEAR THE CONSOLE

    Click Clear to remove the statistics display from the design workspace and the execution console:

Next

The next step is to determine why the Job is failing to perform as expected so that you can correct it.
